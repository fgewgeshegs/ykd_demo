## 今日工作内容

### 1. 调研微信 iLink 协议

- 了解 iLink（智联）是腾讯官方 2026 年通过 OpenClaw 平台开放的**微信个人号 Bot API**
- 协议基于纯 HTTP/JSON，接入域名 `ilinkai.weixin.qq.com`，无封号风险
- 核心流程：扫码登录 → 长轮询收消息 → 回复消息
- 对比了多款开源 SDK（Python/Node.js/Java），最终选择 Java SDK 以匹配现有技术栈

### 2. 搭建 Spring Boot + iLink SDK 项目

- 选用开源 Java SDK：[wechat-ilink-sdk-java](https://github.com/lith0924/wechat-ilink-sdk-java) v2.3.3（MIT 协议）
- 完成项目架构设计：
  - `ILinkConfigProperties` — 配置类，支持 `application.yaml` 灵活配置
  - `ILinkBotService` — 核心服务：QR 扫码登录、消息长轮询、固定回复
  - `BotController` — REST 管理 API（状态查询、二维码获取、重启）
- 功能特性：自动重连、心跳保活、context_token 自动管理

### 3. 调试与测试

- 编译调通，修复 SDK API 兼容性问题
- 解决端口冲突问题
- 成功扫码登录，完成端到端消息收发测试

## 遇到的问题与解决

| 问题 | 解决方案 |
|------|----------|
| SDK API 方法名不匹配（`getNickName()`/`getId()` 不存在） | 反编译 SDK jar 确认实际 API，改为 `getUserId()`/`getMessage_id()` |
| 测试时 Bot 自动启动导致挂起 | 增加 `auto-start` 开关，测试环境禁用 |
| 端口 8080 被占用 | `netstat` 定位进程后 `Stop-Process` 释放端口 |
