# 上下文功能详解

## 一、设计理念

**一个队列存所有**——文字、语音、图片、助手回复，全部放入同一个消息队列。CDN 下载参数直接嵌入 `Message` 记录，不维护独立的媒体存储。

核心理念：**发一条，存一条**。不管用户发什么类型的消息，接收后立刻存入上下文，保证 LLM 能拿到完整的对话历史。

---

## 二、核心数据结构

### 2.1 Message（消息记录）

```java
public record Message(
    String role,              // "user" 或 "assistant"
    String content,           // 展示给 LLM 的文本内容
    String mediaEncryptParam, // 媒体 CDN 加密参数（语音/图片有值，文字为 null）
    String mediaAesKey,       // 媒体 CDN 解密密钥
    String mediaUrl           // 媒体 URL（图片时有值）
) {}
```

纯文字消息用简洁构造器：
```java
new Message("user", "今天天气怎么样")  // CDN 字段自动为 null
```

带媒体的消息：
```java
new Message("user", "[语音]画一只猫", "enc_xxx", "key_yyy", null)
new Message("user", "[图片]",        "enc_xxx", "key_yyy", "https://image.url")
```

`hasMedia()` 方法判断是否有 CDN 参数：
```java
// 有 CDN 参数 = 可重新下载媒体
message.hasMedia() → mediaEncryptParam != null && mediaAesKey != null
```

### 2.2 存储结构

```
InMemoryContextStore / RedisContextStore
  └─ store: userId → Deque<Message>（按时间正序排列）
       │
       ├─ Message("user", "我叫小明")
       ├─ Message("assistant", "你好小明！有什么可以帮你的？")
       ├─ Message("user", "[语音]画一只猫",        "enc1", "key1", null)
       ├─ Message("assistant", "好的，正在为你生成一只猫...")
       ├─ Message("user", "[图片]",               "enc2", "key2", "https://...")
       ├─ Message("assistant", "这是一只橘猫，躺在窗台上晒太阳")
       ├─ Message("user", "[语音]介绍一下你自己",   "enc3", "key3", null)
       └─ ...
```

- 每条 Message 自带 CDN 参数，不需要额外查询
- 最多保留 **50 条/用户**，超出自动淘汰最早的消息
- 发给 LLM 时取最近 **20 条**（ChatService 控制）
- `content` 直接作为 LLM 的对话历史，`role` 区分用户/助手

## 三、写入流程（WechatMessageService.saveMessageToContext）

位置：`WechatMessageService.pollLoop()` — 消息构建后、路由前调用。

```java
// 每条消息路由前统一存入上下文
saveMessageToContext(wechatMsg);
```

### 3.1 文字消息

```
用户发"今天天气怎么样"
  → contextStore.append(userId, "user", "今天天气怎么样")
  → Message("user", "今天天气怎么样")
```

### 3.2 语音消息

```
用户发语音"画一只猫"
  → WechatMessage(type=VOICE, voiceText="画一只猫",
                  voiceEncryptQueryParam="enc_xxx", voiceAesKey="key_yyy")

  → contextStore.addVoiceMedia(userId, "enc_xxx", "key_yyy", "画一只猫")
     → appendMedia → Message("user", "[语音]画一只猫", "enc_xxx", "key_yyy", null)

  → contextStore.append(userId, "user", "[语音]画一只猫")
     → Message("user", "[语音]画一只猫")
```

注意：语音消息在对话历史中会存**两条**：
- 一条带 CDN 参数（`addVoiceMedia`）——用于后续重新下载
- 一条纯文本（`append`）——作为对话历史发给 LLM

实际上 `addVoiceMedia` 内部也是调 `appendMedia`，也写入了队列。所以队列中是一条带 CDN 的 Message。`append` 又追加了一条纯文本的——这里存了两条是因为 `addVoiceMedia` 内部已经追加到队列了。实际上这个设计有重复，但因为 `addVoiceMedia` 追加的 Message 已经带了 `content`，LLM 已经能读到 `[语音]画一只猫`。

### 3.3 图片消息

```
用户发一张图片
  → WechatMessage(type=IMAGE, encryptQueryParam="enc_xxx", aesKey="key_yyy",
                  imageUrl="https://...")

  → contextStore.addImageMedia(userId, "enc_xxx", "key_yyy", "https://...")
     → Message("user", "[图片]", "enc_xxx", "key_yyy", "https://...")

  → contextStore.append(userId, "user", "[图片]")
     → Message("user", "[图片]")
```

### 3.4 助手回复

```
ChatService.chat() 内部：
  → contextStore.append(userId, "assistant", replyText)
  → Message("assistant", "今天北京晴，25°C")
```

### 3.5 助手语音回复（TTS 生成的 MP3）

```
VoiceHandler — TTS 成功后：
  → contextStore.saveAssistantAudio(userId, mp3Bytes)
  → 单独存储（不在对话队列中）

内存实现：ConcurrentHashMap<String, byte[]> 直接存字节
Redis实现：ctx:{userId}:msgs:audio → base64 编码，TTL 7天
```

## 四、读取流程

### 4.1 多轮对话 —— ChatService

ChatService 每次调用 LLM 前取最近 20 条历史：

```java
// ChatService.chat()
List<Message> history = contextStore.getHistory(userId, 20);
String reply = llmClient.chat(userId, message, history);

// 对话完成后追加
contextStore.append(userId, "user", message);        // 用户消息
contextStore.append(userId, "assistant", reply);      // 助手回复
```

LLM 看到的对话历史示例：
```
user: 我叫小明
assistant: 你好小明！有什么可以帮你的？
user: [语音]画一只猫
assistant: 好的，正在为你生成...
user: [图片]
assistant: 这是一只橘猫...
user: 刚才那只猫是什么颜色？
```

LLM 能通过 `[语音]` 和 `[图片]` 标记知道用户发过语音和图片，理解完整上下文。

### 4.2 快速查询最近媒体

**最近一张图片**（VisionHandler 用于"分析一下这张图"）：
```java
String[] img = contextStore.getLastImage(userId);
// → [encryptParam, aesKey, url]

// 内部实现：反向扫描队列，找第一条 [图片] 且 hasMedia() 的 Message
// 取到 CDN 参数后 → downloadMedia(enc, key) → 重新下载图片
```

**最近一条语音**：
```java
String[] voice = contextStore.getLastVoice(userId);
// → [encryptParam, aesKey, text]
```

### 4.3 查询所有历史媒体

**所有语音**（可遍历历史语音重新下载）：
```java
List<String[]> voices = contextStore.getVoiceMediaList(userId);
// → [[enc1, key1, "画一只猫"], [enc2, key2, "今天天气"], ...]
```

**所有图片**：
```java
List<String[]> images = contextStore.getImageMediaList(userId);
// → [[enc1, key1, "https://..."], [enc2, key2, "https://..."], ...]
```

### 4.4 取回助手语音回复

```java
byte[] mp3 = contextStore.getAssistantAudio(userId);
// → 助手最后一次 TTS 生成的 MP3 字节
```

## 五、双存储实现

### 5.1 InMemoryContextStore（默认）

```
数据结构: ConcurrentHashMap<String, Deque<Message>>
容量: 50条/用户（FIFO淘汰）
线程安全: ConcurrentHashMap + synchronized(Deque)
生命周期: 进程重启即丢失

附加存储:
  - assistantAudioStore: ConcurrentHashMap<String, byte[]>  // 助手语音MP3
```

自动启用条件：`context.redis.enabled=false` 或未配置

### 5.2 RedisContextStore

```
Key:  ctx:{userId}:msgs      → Redis LIST（JSON序列化的Message数组）
TTL:  7天（可配置 context.redis.ttl-days）
操作: RPUSH追加 / LTRIM限长 / EXPIRE设TTL / LRANGE取历史

附加Key:
  ctx:{userId}:msgs:audio     → Redis STRING（base64编码的MP3，TTL 7天）
```

自动启用条件：`context.redis.enabled=true`

```properties
# 切换配置
context.redis.enabled=true
context.redis.ttl-days=7
context.redis.max-messages=50
spring.data.redis.host=localhost
spring.data.redis.port=6379
```

### 5.3 JSON 序列化格式

Message 在 Redis 中存储为 JSON：
```json
{
  "role": "user",
  "content": "[语音]画一只猫",
  "mediaEncryptParam": "enc_xxx",
  "mediaAesKey": "key_yyy",
  "mediaUrl": null
}
```

## 六、完整上下文生命周期

```
═══════════════════════════════════════════════════════════════
一、消息到达
═══════════════════════════════════════════════════════════════

① pollLoop 收到消息
   → buildWechatMessage() 构建消息对象

② saveMessageToContext(msg)
   ├─ TEXT:   append("user", text)
   ├─ VOICE:  addVoiceMedia(enc, key, text) + append("user", "[语音]text")
   ├─ IMAGE:  addImageMedia(enc, key, url)  + append("user", "[图片]")
   └─ 每条消息都存入对话队列，CDN参数嵌入Message

③ messageRouter.route(msg) → Handler处理

④ 如果是语音消息: VoiceHandler ASR转文字 → 意图分类 → 路由到具体Handler
   如果是文字消息: MessageRouter 意图分类 → 路由到具体Handler

═══════════════════════════════════════════════════════════════
二、回复生成
═══════════════════════════════════════════════════════════════

⑤ ChatService.chat(userId, text)
   → getHistory(userId, 20) ← 从队列取最近20条Message
   → 遍历Message，用 role+content 构建LLM上下文
   → llmClient.chat(text, history) → 生成回复

⑥ append("user", text) + append("assistant", reply)
   ← 对话内容追加到队列

⑦ 如果触发TTS:
   → VoiceService.synthesize(reply) → MP3字节
   → saveAssistantAudio(userId, mp3) ← MP3单独存储
   → sendFileMessage / sendVoiceMessage → 发送给用户

═══════════════════════════════════════════════════════════════
三、后续引用
═══════════════════════════════════════════════════════════════

⑧ 用户说"分析一下刚才那张图"
   → VisionHandler: getLastImage(userId)
   → 反向扫描队列 → 找到 "[图片]" Message → 取CDN参数
   → downloadMedia(enc, key) → 重新下载 → 分析

⑨ 用户说"播放上次的语音回复"
   → getAssistantAudio(userId) → 取回MP3字节
```

## 七、关键特性总结

| 特性 | 实现 |
|------|------|
| 统一存储 | 文字/语音/图片/回复 全部存入同一队列 |
| CDN嵌入 | 媒体参数直接挂在 Message 记录上，不另建索引 |
| 容量限制 | 50条/用户，超出FIFO淘汰 |
| LLM上下文 | getHistory() 取最近20条发给 LLM |
| 媒体重下载 | getLastImage/getLastVoice 从历史中取CDN参数 → downloadMedia |
| 历史遍历 | getVoiceMediaList/getImageMediaList 遍历全部媒体 |
| 助手语音 | saveAssistantAudio/getAssistantAudio 单独存取 MP3 |
| 双存储 | 内存默认，Redis一键切换，接口完全相同 |
| 会话清除 | clear(userId) 删除该用户所有数据 |
| 线程安全 | ConcurrentHashMap + synchronized(Deque) |
