## 实现了什么

从零搭建微信 AI 助手机器人，交付三项核心能力：

1. **微信 iLink Bot 接入** — 调研 iLink 协议，集成开源 SDK，实现扫码登录、长轮询消息收发、自动重连保活。
2. **AI 多模态对话** — 接入百炼 LLM 实现文本对话；接入 qwen-vl-max 实现图片识别（微信 CDN 下载 → 多模态分析 → 返回描述）；接入 qwen-image-2.0 实现文生图。
3. **图片生成升级为媒体文件发送** — 不再返回文本 URL，改为下载图片 → 上传微信 CDN → 以图片消息直接发送。设计了 `ReplyMessage` 模型替代 String 返回值，全链路 12 个文件适配。

## 遇到的问题 & 解决

1. **生图 API "url error"** — qwen-image-2.0 与 wanx-v1 协议不兼容。对比官方文档，改 base-url 和请求/响应 JSON 结构后解决。
2. **图片生成返回 URL 而非图片** — 根因是 Handler 只返回 String，发送端无法区分文本和图片。新建 `ReplyMessage` 模型（text/image 双工厂方法），Handler 完成生成→下载→上传 CDN 全流程，发送端根据类型选择 sendTextMessage 或 sendImageMessage。
3. **Git 空分支 push 报错** — 无 commit 记录被拒，`git commit -m "init"` 后推送成功。

## 收获

1. 完整走通微信 CDN 媒体加密链路（AES-ECB 加解密 → getUploadUrl → CDN 直传）。
2. 返回值类型的设计影响整个调用链的信息传递能力——早期投入模型类比后期打补丁划算。
3. API 协议排查先对比文档定位差异，比盲目试错高效。
