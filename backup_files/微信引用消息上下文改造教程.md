# 微信引用消息：让 LLM 知道"你在回复哪句话"

## 问题

```
机器人回："图里有一只橘猫坐在沙发上"

你长按这条消息 → 引用 → 打字："它在室内还是室外？"

当前：LLM 只看到 "它在室内还是室外？" → 不知道"它"指什么
```

微信的"引用"功能——长按一条消息再回复——在 SDK 里叫 `RefMessage`。之前的代码完全忽略了这个信息。

## 目标

```
机器人回："图里有一只橘猫坐在沙发上"

你引用 ↑ → 打字："它在室内还是室外？"

LLM 看到:
  [用户引用了: "图里有一只橘猫坐在沙发上"]
  它在室内还是室外？
  → "从图片来看，猫在室内"  ← 答对了
```

---

## 一、整体思路

SDK 的 `MessageItemDto` 有个 `getRefMessage()` 方法。当用户引用了一条消息后发消息，它返回一个 `RefMessage` 对象：

```java
RefMessage ref = item.getRefMessage();
// ref.getTitle()        → 被引用消息的显示文字
// ref.getMessageItem()  → 被引用消息的完整 DTO（可以是文字/图片/语音）
```

我们的改动链路：

```
pollLoop: item.getRefMessage() != null
  → 提取引用文字 → setRefMessageText()
  → AIChatHandler.handle()
    → buildFullText() 把引用拼到消息前面
    → chatService.chat(userId, "[用户引用了: \"...\"]\n实际消息")
    → LLM 看到了完整上下文
```

**为什么在处理链的前端拼而不是在 ChatService 里拼**：引用是"消息格式"层面的东西，属于 Handler 的职责。ChatService 不应该知道微信协议细节。

## 三、逐个详解

### 1. WechatMessage — 加一个字段

```java
/**
 * 引用消息的文字内容（用户引用了一条消息后发送的消息时有效）
 * 优先取引用消息的文本，文本为空时取 title
 */
private String refMessageText;
```

用 `refMessageText` 存提取后的纯文字，后续 Handler 直接用就行。

### 2. WechatMessageService — 提取引用文字

在 pollLoop 里，每种消息类型解析完之后，统一调 `extractRefMessage()`：

```java
// 1.5 提取引用消息文字
extractRefMessage(item, wechatMsg);

// 2. 交由 MessageRouter 路由处理
```

`extractRefMessage` 的逻辑是按优先级提取：

```java
private void extractRefMessage(MessageItemDto item, WechatMessage wechatMsg) {
    RefMessage ref = item.getRefMessage();
    if (ref == null) return;

    MessageItemDto refItem = ref.getMessageItem();

    // ① 优先：引用的是文字消息 → 直接取文本
    if (refItem != null && refItem.getText() != null) {
        wechatMsg.setRefMessageText(refItem.getText());
        return;
    }

    // ② 次选：引用的是语音消息 → 取微信转写
    if (refItem != null && refItem.getVoice() != null
            && refItem.getVoice().getText() != null) {
        wechatMsg.setRefMessageText(refItem.getVoice().getText());
        return;
    }

    // ③ 降级：取 title（被引用消息的显示摘要）
    if (ref.getTitle() != null) {
        wechatMsg.setRefMessageText(ref.getTitle());
    }
}
```

**为什么三级降级**：
- ① 文字消息直接取文本，最准确
- ② 语音消息用微信转写，勉强可用
- ③ `title` 是微信自动生成的摘要（比如图片消息的 title 可能是"[图片]"），聊胜于无

### 3. AIChatHandler — 拼到消息前面

```java
@Override
public String handle(WechatMessage message) {
    // 拼入引用消息内容
    String fullText = buildFullText(message);
    String reply = chatService.chat(message.getUserId(), fullText);
    ...
}

private String buildFullText(WechatMessage message) {
    String refText = message.getRefMessageText();
    if (refText == null || refText.isEmpty()) {
        return message.getText();  // 没引用，原样返回
    }
    return "[用户引用了: \"" + refText + "\"]\n" + message.getText();
}
```

效果：
```
[用户引用了: "图里有一只橘猫坐在沙发上"]
它在室内还是室外？
```

**为什么用中文标签而不是用 JSON 格式**：LLM 对自然语言的提示理解最好。`[用户引用了: "..."]` 是人读的格式，LLM 也能正确理解为"前面是上下文，后面才是问题"。

### 4. VoiceHandler — 传递引用

语音消息也可能被引用发送。VoiceHandler 构造虚拟 TEXT 消息时，把 `refMessageText` 传过去：

```java
textMsg.setRefMessageText(message.getRefMessageText());  // 传递引用消息
```

这样语音 + 引用的场景也能正常工作。

## 四、完整数据流

```
══════════════════════════════════════════════════════════
机器人之前回："图里有一只橘猫坐在沙发上"

用户长按 ↑ → 引用 → 打字："它在室内还是室外？"
══════════════════════════════════════════════════════════

1. pollLoop:
   item.isText() == true
   item.getText() → "它在室内还是室外？"
   item.getRefMessage() != null
     → ref.getMessageItem().getText() → "图里有一只橘猫坐在沙发上"
     → wechatMsg.refMessageText = "图里有一只橘猫坐在沙发上"
     ↓
2. MessageRouter: TEXT → AIChatHandler
     ↓
3. AIChatHandler:
   buildFullText(message):
     refMessageText = "图里有一只橘猫坐在沙发上"
     → "[用户引用了: \"图里有一只橘猫坐在沙发上\"]\n它在室内还是室外？"
   chatService.chat("user123", fullText)
     ↓
4. ChatService:
   contextStore.getHistory("user123") → [历史...]
   llmClient.chat(fullText, history)
     → LLM 看到:
       messages: [
         system,
         ...history...,
         user: "[用户引用了: \"图里有一只橘猫坐在沙发上\"]\n它在室内还是室外？"
       ]
     → "从图片来看，猫在室内，背景是客厅环境。"
   contextStore.append("user", fullText)
   contextStore.append("assistant", "从图片来看...")
     ↓
5. 用户看到回复 ✓
```

---

## 五、为什么这样设计

**为什么在 Handler 层拼而不是在 ChatService 拼**：职责分离。Handler 负责微信消息格式的适配（引用是微信的概念），ChatService 只看到纯文本。如果以后加其他渠道（Telegram/Discord），引用格式不同，只需要换 Handler，ChatService 不用动。

**为什么不直接传 RefMessage 对象给 LLM**：LLM 只认文字。RefMessage 里的 `getMessageItem()` 可能是个图片/语音 DTO，LLM 看不懂。提取成纯文字是最稳妥的做法。

**为什么引用的是图片消息时 title 通常只是 "[图片]"**：微信的引用标题是客户端生成的摘要。引用图片消息时，微信只显示 "[图片]"，对 LLM 几乎没用。但这是微信的限制，不是我们的问题。如果用户引用的是之前机器人回复的图片分析文字，`getText()` 能拿到完整分析结果，LLM 就能理解。

**引用和 ContextStore 的关系**：引用是"当前这一条消息的附加信息"，ContextStore 是"整个会话的记忆"。它们不互斥——引用内容会作为当前消息的一部分存进 ContextStore，后续对话的 LLM 能同时看到历史对话和当前引用。
