# 图片生成：从返回文字链接改成直接发图片

## 问题

```
你发："画一只猫"
机器回："已为您生成图片：https://xxx.com/cat.png"  ← 文字链接，不是图
```

## 目标

```
你发："画一只猫"
机器回：[一张猫的图片]  ← 图片消息
```

---

## 一、整体流程

```
① AI 生成图片 → 返回 URL（图片网址）
     ↓
② HttpClient GET → byte[] 到内存（下载）
     ↓
③ wechatClient.uploadMedia(byte[]) → 上传微信 CDN，拿到凭证
     ↓
④ wechatClient.sendImageMessage(token, 凭证) → 微信发送图片消息
     ↓
⑤ 用户微信根据凭证去 CDN 拉图 → 看到图片
```

**为什么用 URL 而不是 base64：** base64 把几百 KB 的图片塞进 JSON 里传，响应又大又慢，而且比原图多占 33% 体积。URL 方式走专门的二进制通道下载，更快更稳。

---

## 二、改哪些文件（5 个）

| 文件 | 改什么 |
|---|---|
| `WechatMessage` | +1 字段 `contextToken` |
| `WechatMessageService` | +1 行 `setContextToken()` |
| `WechatILinkClient` | +2 方法 `uploadMedia()` `sendImageMessage()` |
| `ImageGenerationHandler` | 核心管线：生成→下载→上传 CDN→发图 |

**没动：** 接口、Router、其他 3 个 Handler、ImageGenerationService、ImageGenerationTool、ImageClient。

## 三、逐个详解

### 1. WechatMessage — 加一个字段

```java
private String contextToken;  // 会话令牌
```

Handler 发图需要 token，pollLoop 有但以前没传进来。

### 2. WechatMessageService — 加一行

```java
wechatMsg.setContextToken(contextToken);  // 把 token 塞进 message
```

### 3. WechatILinkClient — 加两个方法

```java
// 上传文件到微信 CDN（mediaType: 1=图片）
// SDK 内部自动：AES加密 → 申请上传凭证 → PUT到CDN
// 返回 MediaInfo{encryptQueryParam, aesKey, fileSize}
public ILinkClient.MediaInfo uploadMedia(int mediaType, String toUserId, byte[] fileData)

// 发送图片消息
public void sendImageMessage(String toUserId, String contextToken, ILinkClient.MediaInfo mediaInfo)
```

SDK 已有，包一层异常处理，风格跟已有的 `sendTextMessage` 一致。

### 4. ImageGenerationHandler — 核心（完整代码）

```java
public String handle(WechatMessage message) {
    // ── ① 调 Service 生成，拿到 "已为您生成图片：URL" ──
    String result = imageGenerationService.generate(message.getText());
    // 切掉前缀，提取纯 URL
    String imageUrl = result.startsWith("已为您生成图片：")
        ? result.substring("已为您生成图片：".length()) : null;
    if (imageUrl == null) return "失败文案";

    // ── ② 下载图片到内存 ──
    byte[] imageBytes = downloadImage(imageUrl);  // HttpClient GET
    if (imageBytes == null) return "失败文案";

    // ── ③ 上传微信 CDN ──
    ILinkClient.MediaInfo mediaInfo = wechatClient.uploadMedia(1, message.getUserId(), imageBytes);
    if (mediaInfo == null) return "失败文案";

    // ── ④ 发送图片消息 ──
    wechatClient.sendImageMessage(
        message.getUserId(),        // 发给谁
        message.getContextToken(),  // 会话令牌
        mediaInfo                   // CDN 凭证
    );
    return null;  // 不发文字
}

// 下载方法
private byte[] downloadImage(String imageUrl) {
    HttpRequest request = HttpRequest.newBuilder()
        .uri(URI.create(imageUrl)).timeout(Duration.ofSeconds(60)).GET().build();
    HttpResponse<byte[]> response = httpClient.send(request, BodyHandlers.ofByteArray());
    return (response.statusCode() >= 200 && response.statusCode() < 300)
        ? response.body() : null;
}
```

**每一步失败都 return 降级文字**，确保用户总能看到反馈。

## 四、完整数据流（一次"画一只猫"）

```
1. 用户发"画一只猫"
     ↓
2. pollLoop: new WechatMessage
   msg.userId = "user123"
   msg.text = "画一只猫"
   msg.contextToken = "abc..."           ← 新加的
     ↓
3. MessageRouter → IMAGE_GENERATE → ImageGenerationHandler
     ↓
4. Handler:
   ① imageGenerationService.generate("画一只猫")
      → ImageClient POST 阿里百炼 → 返回图片 URL
      → Service 拼接 "已为您生成图片：https://xxx.png"
      → Handler 切掉前缀，拿到 "https://xxx.png"
   ② HttpClient GET https://xxx.png → byte[400000]
   ③ wechatClient.uploadMedia(1, "user123", bytes)
      → SDK: AES加密 → 拿上传凭证 → PUT到CDN
      → MediaInfo{encryptParam, aesKey, fileSize}
   ④ wechatClient.sendImageMessage("user123", "abc...", mediaInfo)
   ⑤ return null
     ↓
5. pollLoop: reply == null → 跳过
     ↓
6. 用户微信根据凭证拉 CDN → 看到猫 ✓
```

---

## 五、为什么这样设计

**为什么 Handler 自己发图：** 拿到了 contextToken，有能力自理。pollLoop 完全不用感知图片这回事——图发了就 return null，跟没回复一样跳过。

**为什么注入 ImageGenerationService 而不是 ImageClient：** Service 返回 "已为您生成图片：URL"，Handler 切前缀提取 URL。Service 还保留了原来的逻辑，Agent 通道（ImageGenerationTool）继续走它，互不影响。

**为什么 HttpClient 在 Handler 里 new：** 就这一个地方下载图片。项目里 LLMClient、VisionClient 也是各自 new HttpClient，保持一致。

**为什么每一步都 return 降级文字：** 异步轮询不能抛异常——单条失败不能影响后续消息。降级文字确保用户至少看到反馈。
