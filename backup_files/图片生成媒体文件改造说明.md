# 图片生成媒体文件改造说明

## 1. 背景与目标

### 1.1 改造前的问题

原来的图片生成流程：

```
用户发送文字 → ImageClient 调用 AI 模型 → 返回图片 URL
  → ImageGenerationService 拼接为 "已为您生成图片：https://xxx.png"
  → WechatMessageService 以文本消息形式发送
  → 用户收到一段包含 URL 的文字 ❌
```

用户看到的是文字 URL，无法直接在聊天中预览图片。

### 1.2 改造后的目标

```
用户发送文字 → ImageClient 调用 AI 模型 → 返回图片 URL
  → 下载图片字节 → 上传到微信 CDN → 以图片消息形式发送
  → 用户直接在聊天中看到图片 ✓
```

## 3. 各文件改动详解

### 3.1 新建：`ReplyMessage.java`

**路径：** `src/main/java/com/youkeda/exercise/claw/wechat/model/ReplyMessage.java`

**设计思路：** 原来所有 Handler 返回 `String`，调用方（`WechatMessageService`）无法区分"这是一段要发文本的回复"还是"这是一张要发图片的回复"。引入 `ReplyMessage` 模型统一封装两种回复类型。

**核心 API：**

```java
// 工厂方法 — 创建文本回复
ReplyMessage.text("你好！");

// 工厂方法 — 创建图片回复（携带微信 CDN 媒体信息）
ReplyMessage.image(encryptQueryParam, aesKey, fileSize);

// 类型判断
reply.isText();    // true: 有文本且无图片
reply.isImage();   // true: 有 CDN 媒体信息
reply.isEmpty();   // true: 既无文本也无图片
```

**关键设计决策：**

- **构造函数私有 + 静态工厂方法**：`text()` 和 `image()` 明确表达意图，外部无法构造出不一致的状态
- **`isText()` 判断条件**：`text != null && imageEncryptQueryParam == null`。如果同时设置了文本和图片（image 可能有降级文本），以图片为准
- **`isImage()` 判断条件**：仅检查 `imageEncryptQueryParam != null`，不与 text 互斥（因为图片携带的 CDN 信息量更大）

**字段说明：**

| 字段 | 类型 | 用途 |
|---|---|---|
| `text` | `String` | 文本回复内容 |
| `imageEncryptQueryParam` | `String` | 微信 CDN 加密查询参数，发送图片消息时必须 |
| `imageAesKey` | `String` | 微信 CDN AES 解密密钥，发送图片消息时必须 |
| `imageFileSize` | `long` | 上传到 CDN 后的文件大小 |

### 3.2 扩展：`WechatILinkClient.java`

**路径：** `src/main/java/com/youkeda/exercise/claw/wechat/client/WechatILinkClient.java`

**新增方法：**

#### `uploadMedia()`

```java
/**
 * 上传媒体文件到微信 CDN
 * @param mediaType 媒体类型：1=图片, 2=视频, 3=文件, 4=语音
 * @param toUserId  目标用户 ID
 * @param fileData  文件字节数组
 * @return MediaInfo（含 encryptQueryParam、aesKey、fileSize），失败时返回 null
 */
public ILinkClient.MediaInfo uploadMedia(int mediaType, String toUserId, byte[] fileData)
```

**内部流程：**
1. 检查登录状态（未登录直接返回 null）
2. 调用 SDK `client.uploadMedia(credentials, mediaType, toUserId, fileData)`
3. SDK 内部自动完成：生成 fileKey + AES 密钥 → AES-ECB 加密文件 → 计算 MD5 → 调用微信 `/ilink/bot/getuploadurl` 获取上传 URL → 上传到微信 CDN → 返回 `MediaInfo`

**返回值 `ILinkClient.MediaInfo`：**

| 字段 | 说明 |
|---|---|
| `encryptQueryParam` | CDN 加密查询参数，约 2000+ 字符的加密串 |
| `aesKey` | Base64 编码的 AES 密钥（hex 字符串的 base64） |
| `fileSize` | 加密后的文件大小 |

#### `sendImageMessage()`

```java
/**
 * 发送图片消息
 * @param toUserId     目标用户 ID
 * @param contextToken 会话令牌（从接收消息时获取）
 * @param mediaInfo    媒体信息（从 uploadMedia 获取）
 */
public void sendImageMessage(String toUserId, String contextToken, ILinkClient.MediaInfo mediaInfo)
```

**内部流程：**
1. 检查登录状态
2. 调用 SDK `client.sendImageMessage(credentials, toUserId, contextToken, mediaInfo)`
3. SDK 构造 `SendMessageRequest`（`messageType=2`, `itemList[0].type=2` 即图片），填充 CDN 媒体实体

**与原有方法的风格一致性：** 所有新增方法遵循 `sendTextMessage()` 的模式——未登录时 warn 并返回、异常时 catch + log error（不向上抛），确保不会因单条消息发送失败导致轮询线程退出。

### 3.3 精简：`ImageGenerationService.java`

**路径：** `src/main/java/com/youkeda/exercise/claw/ai/image/ImageGenerationService.java`

**改动：** 移除 `FALLBACK_MESSAGE` 常量和文本拼接逻辑，`generate()` 返回纯 URL 或 null。

```java
// 改造前
public String generate(String prompt) {
    // ...
    return "已为您生成图片：" + imageUrl;  // 拼接了文本前缀
}

// 改造后
public String generate(String prompt) {
    // ...
    return imageUrl;  // 返回纯 URL，null 表示失败
}
```

**为什么这样改：** Service 层的职责是"生成图片并返回 URL"，不应该关心"这个 URL 最终以什么形式呈现给用户"。文本拼接逻辑上移到调用方：

- **WeChat 图片消息通道**（`ImageGenerationHandler`）：不需要文本前缀，直接以图片形式发送
- **Agent 文本通道**（`ImageGenerationTool`）：需要文本前缀，在 Tool 层自行拼接

### 3.4 适配：`ImageGenerationTool.java`

**路径：** `src/main/java/com/youkeda/exercise/claw/agent/ImageGenerationTool.java`

**改动：** 因为 `ImageGenerationService.generate()` 现在返回 null 表示失败，Tool 需要自行处理 null 并拼接文本前缀。

```java
@Override
public String execute(AgentContext context) {
    String imageUrl = imageGenerationService.generate(context.getMessage());
    if (imageUrl == null) {
        return "抱歉，图片生成失败，请稍后再试。";
    }
    return "已为您生成图片：" + imageUrl;
}
```

**为什么保留文本前缀：** `ImageGenerationTool` 属于 Agent 体系，Agent 通过 `AgentExecutor.execute()` 返回 `String`，这个字符串会作为文本消息发送。所以此处需要保留友好的文本提示。

### 3.5 核心重写：`ImageGenerationHandler.java`

**路径：** `src/main/java/com/youkeda/exercise/claw/wechat/handler/ImageGenerationHandler.java`

**这是本次改造最核心的改动。** Handler 不再简单地返回文本字符串，而是完成完整的"生成→下载→上传→返回 CDN 信息"管线。

**新增依赖：**

```java
private final ImageGenerationService imageGenerationService;  // 原有
private final WechatILinkClient wechatClient;                 // 新增：上传 CDN
private final HttpClient httpClient;                          // 新增：下载图片
```

**完整流程（`handle()` 方法）：**

```
Step 1: imageGenerationService.generate(prompt)
        → 调用 AI 模型生成图片
        → 返回图片 URL（如 https://dashscope.aliyuncs.com/.../xxx.png）
        → null → 降级返回 ReplyMessage.text(FALLBACK_TEXT)

Step 2: downloadImage(imageUrl)
        → HttpClient 发送 GET 请求
        → 检查 HTTP 2xx 状态码
        → 返回 byte[]
        → null/empty → 降级返回 ReplyMessage.text(FALLBACK_TEXT)

Step 3: wechatClient.uploadMedia(1, userId, imageBytes)
        → mediaType=1 表示图片
        → SDK 自动完成加密+上传 CDN
        → 返回 ILinkClient.MediaInfo{encryptQueryParam, aesKey, fileSize}
        → null → 降级返回 ReplyMessage.text(FALLBACK_TEXT)

Step 4: ReplyMessage.image(encryptQueryParam, aesKey, fileSize)
        → 返回图片 ReplyMessage，CDN 信息由 WechatMessageService 消费
```

**下载图片实现（`downloadImage()`）：**

```java
private byte[] downloadImage(String imageUrl) {
    HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create(imageUrl))
            .timeout(Duration.ofSeconds(60))
            .GET()
            .build();

    HttpResponse<byte[]> response = httpClient.send(request,
            HttpResponse.BodyHandlers.ofByteArray());

    if (response.statusCode() >= 200 && response.statusCode() < 300) {
        return response.body();
    }
    return null;
}
```

**异常处理策略：** 每一步失败都降级为文本回复（"抱歉，图片生成失败，请稍后再试。"），确保用户总能看到反馈，不会因中间环节失败而静默。

### 3.11 桥接：`SimpleAgentExecutor.java`

**路径：** `src/main/java/com/youkeda/exercise/claw/agent/SimpleAgentExecutor.java`

**改动：** 因为 `AgentExecutor.execute()` 返回 `String`，而 `MessageRouter.route()` 现在返回 `ReplyMessage`，需要增加转换逻辑。

```java
@Override
public String execute(AgentContext context) {
    // ...
    ReplyMessage reply = messageRouter.route(wechatMsg);
    return replyToString(reply);
}

private String replyToString(ReplyMessage reply) {
    if (reply == null || reply.isEmpty()) {
        return null;
    }
    if (reply.isImage()) {
        return "已为您生成图片";  // Agent 文本接口无法发送图片，返回友好提示
    }
    return reply.getText();
}
```

**设计考量：** Agent 体系是纯文本通道，无法发送图片消息。当 ImageGenerationHandler 返回图片 Reply 时，此处提取为友好的文本提示。

### 3.12 发送端：`WechatMessageService.java`

**路径：** `src/main/java/com/youkeda/exercise/claw/wechat/service/WechatMessageService.java`

**改动（pollLoop 中的发送逻辑）：**

```java
// 改造前
String reply = messageRouter.route(wechatMsg);
if (reply != null && !reply.isEmpty()) {
    wechatClient.sendTextMessage(fromUserId, contextToken, reply);
}

// 改造后
ReplyMessage reply = messageRouter.route(wechatMsg);
if (reply != null && !reply.isEmpty()) {
    if (reply.isImage()) {
        // 图片回复：构造 MediaInfo，调用 sendImageMessage
        ILinkClient.MediaInfo mediaInfo = new ILinkClient.MediaInfo(
                reply.getImageEncryptQueryParam(),
                reply.getImageAesKey(),
                reply.getImageFileSize());
        wechatClient.sendImageMessage(fromUserId, contextToken, mediaInfo);
    } else {
        // 文本回复：保持原有 sendTextMessage 逻辑
        wechatClient.sendTextMessage(fromUserId, contextToken, reply.getText());
    }
}
```

**关键点：** `fromUserId` 和 `contextToken` 来自 pollLoop 中的接收消息上下文。`contextToken` 是微信会话令牌，必须在回复时原样带回，否则消息无法送达。

## 5. 架构设计决策

### 5.1 为什么新建 `ReplyMessage` 而不是用特殊字符串标记？

| 方案 | 优点 | 缺点 |
|---|---|---|
| 特殊前缀 `[IMAGE:...]` | 改动小 | 字符串解析脆弱，字段转义困难，类型不安全 |
| **`ReplyMessage` 模型** ✅ | 类型安全，编译期检查，扩展性好 | 需要改接口，影响所有 Handler |
| Handler 直接发送 | 改动集中 | 破坏分层，Handler 需要 contextToken，职责过重 |

选择 `ReplyMessage` 模型——虽然改动面大，但类型安全、清晰、后续扩展（如视频、文件）只需增加字段和工厂方法。

### 5.2 为什么 `ImageGenerationService` 不直接返回字节？

Service 层返回 URL，下载逻辑放在 Handler 层：

- **单一职责**：Service 负责调用 AI 模型获取结果，不应关心"图片如何传输"
- **可测试性**：URL 比字节数组更容易在单元测试中验证
- **灵活性**：不同调用方可以自行决定如何消费 URL（Agent 通道不需要下载）

### 5.3 为什么降级返回文本而不是抛异常？

微信消息系统是异步轮询模式——单条消息处理失败不应影响后续消息。所有异常在 Handler 内 catch 并返回友好文本降级，确保：

- 用户总能看到反馈
- 轮询线程不会因单次失败退出
- 后续消息正常处理

### 5.4 CDN 上传链路的安全性

SDK 的 `uploadMedia()` 内部：

1. 随机生成 fileKey（UUID）和 256-bit AES 密钥
2. 使用 AES-ECB 加密文件内容
3. 向微信 `/ilink/bot/getuploadurl` 请求临时上传凭证
4. 将加密数据 PUT 到微信 CDN
5. 返回的 `encryptQueryParam` + `aesKey` 构成下载凭证

微信客户端收到图片消息后，用相同的参数从 CDN 下载并解密，确保传输安全。
