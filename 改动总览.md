## 一、新增 context 包 — 会话上下文存储

### 文件
- `context/Message.java` — 新增
- `context/ContextStore.java` — 新增（接口）
- `context/InMemoryContextStore.java` — 新增（内存实现）

### 接口
```java
List<Message> getHistory(userId, maxMessages);     // 取最近 N 条历史
void append(userId, role, content);                 // 追加消息
void setLastImage(userId, encryptParam, aesKey);    // 存图片 CDN 参数
String[] getLastImage(userId);                      // 取图片 CDN 参数
void setLastImageUrl(userId, url);                  // 存图片 HTTP 地址
String getLastImageUrl(userId);                     // 取图片 HTTP 地址
void clear(userId);                                 // 清除
```

### 实现要点
- `ConcurrentHashMap<userId, Deque<Message>>`，单用户上限 50 条
- `synchronized (deque)` 保证线程安全，`List.copyOf()` 返回不可变快照
- 图片 CDN 参数和 HTTP URL 分开存储

## 三、图片分析上下文

### 文件
- `VisionHandler.java` — 重写

### 入口支持四种场景
```
① type=IMAGE                 → 直接发图
② type=TEXT + refImage       → 引用图片
③ type=TEXT + ContextStore最近图片 → "描述一下"不引用
④ (以上都不满足则 return null)
```

### 图片下载优先级（五级降级）
```
① 消息直接附带的 CDN 参数（直接发图）
  → ② 引用消息中的 CDN 参数（引用图片）
    → ③ ContextStore CDN 参数（上次用户发的图）
      → ④ ContextStore HTTP URL（bot 生成的图，从 AI URL 下载）
        → ⑤ 消息携带的 URL
```

### 分析后写入上下文
```java
contextStore.append(userId, "user", "[用户发送了一张图片]");
contextStore.append(userId, "assistant", reply);
```

## 四、微信引用消息提取

### 文件
- `WechatMessage.java` — 新增字段
- `WechatMessageService.java` — 新增 extractRefMessage + 图文合并
- `AIChatHandler.java` — buildFullText 拼入引用上下文

### WechatMessage 新增字段
```java
refMessageText         // 引用消息的文字
refImageEncryptParam   // 引用图片的 CDN 参数
refImageAesKey         // 引用图片的解密密钥
hasAttachedImage       // 图文合并标记
```

### extractRefMessage 逻辑
```
RefMessage 存在时：
  1. 提取图片 CDN 参数（refItem.getImage()）
  2. 提取文字：优先 refItem.getText() → 语音转写 → ref.getTitle()
```

### 图文合并
```
pollLoop 检测相邻 IMAGE+TEXT 或 TEXT+IMAGE → 合并为一条
  → hasAttachedImage=true, type=TEXT
  → 同时提取两个 item 的 refMessage（修复：之前只取 TEXT item）
  → route 时 hasAttachedImage 的 IMAGE 类型消息走意图分类
```

### AIChatHandler 引用格式
```java
// LLM 收到的格式
"用户引用了以下消息：「被引用内容」
请针对这条被引用的消息，执行以下指令：用户指令"
```

## 五、bot 图片存储（CDN 失败规避）

### 文件
- `ImageGenerationHandler.java` — 修改
- `WechatILinkClient.java` — 修改

### 问题
bot 通过微信 CDN 上传的图片，用同样的 CDN 参数下载会失败（"下载媒体失败"）。

### 方案
bot 生成图片后，分两路存：
1. **AI URL**：`contextStore.setLastImageUrl(userId, aiUrl)` — VisionHandler 第④级用 HTTP 下载
2. **CDN 参数**：`contextStore.setLastImage(userId, encryptParam, aesKey)` — 用户发图时的 CDN 参数可以正常下载

### 诊断日志
```java
log.info("消息诊断 [{}/{}] | type=image={} text={} | ref={} | refImage={}", ...);
log.error("媒体下载失败 | encryptQueryParam={} | aesKeyLen={} | error={}", ...);
```

## 六、Bug 修复清单

| # | Bug | 根因 | 修复 |
|---|---|---|---|
| 1 | 引用图片+文字后无反应 | VisionHandler 拒绝 TEXT 类型 | handle() 增加 TEXT+refImage 入口 |
| 2 | "描述一下"返回兜底回复 | VisionHandler 不查 ContextStore | handle() 增加 ContextStore 最近图片入口 |
| 3 | 引用图片后下载失败 | 图文合并时只取 TEXT item 的 refMessage | 同时提取 IMAGE 和 TEXT item |
| 4 | bot 发的图下载失败 | CDN 参数不适用于 bot 自下载 | 存储 AI URL，HTTP 下载 |
| 5 | 引用后 LLM 翻译了上一条而非被引用消息 | 引用格式 `[用户引用了: "..."]` 不够明确 | 改为 `「...」请针对这条被引用消息执行指令` |
