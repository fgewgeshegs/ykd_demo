# Claw Assistant 项目分析文档

> 分析时间：2026-07-22

---

## 一、项目概览

这是一个基于 **Spring Boot 3.3 + Java 21** 的 AI 助手项目，主要通过 **微信 iLink SDK** 接入微信，实现多模态 AI 对话能力。

### 技术栈

| 项目 | 版本 |
|------|------|
| Spring Boot | 3.3.2 |
| Java | 21 |
| 微信 iLink SDK | 2.3.3 |
| Apache Tika | 2.9.2 |
| PDFBox | 2.0.31 |
| POI | 5.2.5 |

### 多模型 AI 能力

| 能力 | 模型 | 用途 |
|------|------|------|
| 文本对话 | DeepSeek v4 Flash | 聊天、意图分类、内容生成 |
| 视觉分析 | 通义千问 VL-Max | 图片理解、OCR |
| 图片生成 | Qwen-Image 2.0 | AI 画图 |
| 语音识别 | Paraformer v2 | ASR 语音转文字 |
| 语音合成 | Qwen3 TTS Flash | TTS 文字转语音 |

## 三、核心功能实现

### 1. 消息路由架构 (MessageRouter)

```
微信消息 → WechatMessageService(轮询) → MessageRouter → 按类型分发
  ├── IMAGE → VisionTool（视觉分析）
  ├── VOICE → VoiceTool（ASR → 意图分类 → 路由）
  ├── FILE  → FileTool（图片→视觉 / 文档→Tika+LLM）
  └── TEXT  → LLMIntentClassifier → 按意图路由
        ├── CHAT → ChatTool
        ├── IMAGE_GENERATE → ImageGenerationTool
        ├── IMAGE_ANALYZE → VisionTool
        ├── VOICE_REPLY → VoiceTool（TTS）
        └── FILE_GENERATE → FileGenerationTool
```

### 2. 多轮对话上下文 (ContextStore)

- 支持 **内存** / **Redis** 两种存储后端切换
- 每条消息（文本/语音/图片/文件）都存入上下文
- `ChatService` 取最近 20 条历史注入 LLM 对话

### 3. 文件处理 (FileTool + FileParseService)

- 使用 **Apache Tika** 进行文档解析和 MIME 检测
- 图片文件 → 视觉模型分析
- 文档（PDF/DOCX/PPTX） → 文本提取 + 嵌入图片分析 + LLM 总结
- 支持反向生成 PDF/DOCX 文件

### 4. 语音完整链路

```
语音消息 → ASR(微信服务端/Paraformer) → 意图分类(有意义+意图) 
  → CHAT → LLM对话 → TTS合成 → MP3文件回复
  → IMAGE_GENERATE → AI画图 → 图片回复
  → 无意义 → "没听清"
```

### 5. CLI 子系统

除了微信接入，还有控制台 CLI 模式，支持 `help`、`status`、`version`、`weather <city>` 等命令。

## 四、完整流程详解

### 流程一：启动流程

```
Spring Boot 启动
  │
  ├── 1. @PostConstruct: WechatILinkClient.init()
  │     ├── ILinkConfig.builder().heartbeatEnabled(false).build()   // 关闭SDK心跳，避免与getUpdates抢锁
  │     ├── ILinkClient.builder().config(config).build()
  │     └── CompletableFuture.runAsync → client.executeLogin()
  │           ├── 返回 http URL → 打印链接扫码
  │           └── 返回 base64 → 写 qrcode.png → 打印路径扫码
  │           └── 轮询 client.isLoggedIn()，最多等 120s
  │
  ├── 2. @PostConstruct: WechatMessageService.start()
  │     ├── 检查 wechat.ilink.enabled，false 则跳过
  │     ├── 等待 wechatClient.isLoggedIn()，最多等 60s
  │     └── 启动 pollThread (daemon) → pollLoop()
  │
  ├── 3. @PostConstruct: 各 Tool 注册到 ToolRegistry
  │     ├── ChatTool.init() → toolRegistry.register(this)
  │     ├── VisionTool.init() → toolRegistry.register(this)
  │     ├── ImageGenerationTool.init() → toolRegistry.register(this)
  │     └── FileGenerationTool.init() → toolRegistry.register(this)
  │
  ├── 4. @PostConstruct: LLMClient.init() → PromptLoader 加载 system-prompt.txt
  ├── 5. @PostConstruct: VisionClient.init() → 加载 vision-system-prompt.txt
  ├── 6. @PostConstruct: ImageClient.init() → 加载 image-system-prompt.txt
  ├── 7. @PostConstruct: VoiceService.init() → 加载 voice-system-prompt.txt
  │
  └── 8. @PostConstruct: Main.init()
        ├── registerCommands() → 遍历所有 CommandHandler，按类名注册
        └── startCli() → 新线程启动控制台 Scanner 循环
```

### 流程二：消息轮询流程 (WechatMessageService.pollLoop)

```
while(running) {
  │
  ├── wechatClient.receiveMessages()
  │     └── client.getUpdates()  ← SDK v2.3.3，内部管理 cursor
  │
  ├── 遍历 WeixinMessage → 遍历 item_list
  │     │
  │     ├── buildWechatMessage(item, fromUserId, contextToken)
  │     │     ├── text_item != null  → type=TEXT, setText()
  │     │     ├── image_item != null → type=IMAGE, setImageUrl/CDN参数
  │     │     ├── voice_item != null → type=VOICE, setVoiceText/CDN参数
  │     │     └── file_item != null  → type=FILE, setFileName/CDN参数
  │     │
  │     ├── saveMessageToContext(wechatMsg)   ← 统一存入上下文
  │     │     ├── TEXT  → contextStore.append(userId, "user", text)
  │     │     ├── VOICE → contextStore.append(userId, "user", "[语音]voiceText", CDN参数)
  │     │     ├── IMAGE → contextStore.append(userId, "user", "[图片]", CDN参数, imageUrl)
  │     │     └── FILE  → contextStore.append(userId, "user", "[文件: name]", CDN参数)
  │     │
  │     ├── wechatClient.startTyping(userId)  ← 显示"正在输入"
  │     │
  │     ├── messageRouter.route(wechatMsg)     ← 核心路由
  │     │
  │     └── sendReply(userId, reply)
  │           ├── IMAGE → wechatClient.sendImageMessage() → 异步 + 重试2次 + 降级文字
  │           ├── FILE  → wechatClient.sendFileMessage()  → 异步
  │           └── TEXT  → wechatClient.sendTextMessage()
  │
  └── Thread.sleep(pollIntervalMs=3000)
}
```

### 流程三：文本消息流程 (TEXT)

```
用户发送文字消息 "画一只猫"
  │
  ├── MessageRouter.route() → message.getType() == TEXT
  │     │
  │     ├── ① LLMIntentClassifier.classify("画一只猫")
  │     │     │
  │     │     ├── LLMClient.chatWithSystemPrompt(CLASSIFICATION_PROMPT, "画一只猫")
  │     │     │     └── POST DeepSeek /chat/completions
  │     │     │           system: "你是一个意图分类器..."
  │     │     │           user: "画一只猫"
  │     │     │
  │     │     ├── LLM 返回: {"intent":"IMAGE_GENERATE"}
  │     │     └── parseIntent() → Intent.IMAGE_GENERATE
  │     │
  │     ├── ② selectHandler(IMAGE_GENERATE) → ImageGenerationTool
  │     │
  │     └── ③ ImageGenerationTool.handle(message)
  │           │
  │           ├── enrichPrompt(userId, "画一只猫")
  │           │     ├── 文本 < 15字 → 需要扩写
  │           │     ├── contextStore.getHistory(userId, 20)  ← 取最近20条对话
  │           │     └── LLMClient.chatWithSystemPrompt(ENRICH_PROMPT, "画一只猫", history)
  │           │           └── 返回扩写后的完整描述
  │           │
  │           ├── ImageGenerationService.generate(扩写描述)
  │           │     └── ImageClient.generateImage(prompt)
  │           │           └── POST DashScope qwen-image-2.0 API → 返回图片 URL
  │           │
  │           ├── contextStore.updateLastMediaUrl(userId, "[图片]", imageUrl)
  │           ├── ImageClient.downloadImage(imageUrl)
  │           ├── contextStore.append(userId, "assistant", "[图片]", null, null, imageUrl)
  │           └── return WechatReply.image(imageBytes)
  │
  └── sendReply → wechatClient.sendImageMessage(userId, imageBytes, fallback)
        └── CompletableFuture.runAsync → client.sendImage()  ← 异步发送，失败重试2次
```

### 流程四：图片消息流程 (IMAGE)

```
用户发送一张图片
  │
  ├── MessageRouter.route() → message.getType() == IMAGE
  │     │
  │     └── VisionTool.handle(message)  ← 直接走视觉分析，不经过意图分类
  │           │
  │           ├── downloadImageAsDataUrl(message)
  │           │     ├── 优先: message CDN 参数 → wechatClient.downloadMedia()
  │           │     ├── 降级: contextStore.findLastByPrefix(userId, "[图片]") → CDN参数
  │           │     ├── 再降级: contextStore 的 mediaUrl → imageClient.downloadImage()
  │           │     └── 最后: message.getImageUrl() 直接URL
  │           │     └── 返回 "data:image/jpeg;base64,..."
  │           │
  │           ├── VisionService.analyze(dataUrl, null)
  │           │     └── VisionClient.analyzeImage(dataUrl, question)
  │           │           └── POST 通义千问 VL-Max（OpenAI 多模态格式）
  │           │
  │           ├── contextStore.append(userId, "user", "[用户发送了一张图片]")
  │           ├── contextStore.append(userId, "assistant", "这是一张...")
  │           └── return WechatReply.text("这是一张...")
  │
  └── sendReply → wechatClient.sendTextMessage()
```

### 流程五：语音消息流程 (VOICE)

```
用户发送语音消息
  │
  ├── MessageRouter.route() → message.getType() == VOICE
  │     │
  │     └── VoiceTool.handle(message)
  │           │
  │           ├── ① getVoiceText(message) — 获取 ASR 文本
  │           │     ├── 优先: message.getVoiceText()（微信服务端识别结果）
  │           │     └── 降级: 下载音频 → voiceService.transcribe(audioBytes, encodeType)
  │           │           └── VoiceClient.asr()
  │           │                 ├── 上传音频到 DashScope 文件服务
  │           │                 ├── 创建 Paraformer ASR 任务（异步）
  │           │                 └── 轮询任务状态（指数退避: 2s→4s→8s→16s→32s），最多3次重试
  │           │
  │           ├── ② voiceService.classifyVoiceIntent(voiceText) — 一次 LLM 调用完成两件事
  │           │     └── LLMClient.chatWithSystemPrompt(voiceIntentPrompt, voiceText)
  │           │           返回: {"meaningful": true, "intent": "CHAT"}
  │           │     ├── meaningful=false → 回复"没听清"
  │           │     └── meaningful=true → 取 intent
  │           │
  │           ├── ③ routeByIntent(intent, voiceText, message)
  │           │     │
  │           │     ├── CHAT → handleChatWithTts()
  │           │     │     ├── chatTool.handle(textMessage) → ChatService → LLM 对话
  │           │     │     ├── 如果 TTS 启用:
  │           │     │     │     ├── voiceService.synthesize(replyText)
  │           │     │     │     │     └── VoiceClient.tts()
  │           │     │     │     │           ├── POST DashScope CosyVoice TTS
  │           │     │     │     │           ├── 返回音频 URL → 下载字节
  │           │     │     │     │           └── 解析 WAV header → 时长/采样率
  │           │     │     │     └── return WechatReply.file(bytes, "AI语音回复.mp3", replyText)
  │           │     │     └── 如果 TTS 失败 → 降级为文本回复
  │           │     │
  │           │     ├── IMAGE_GENERATE → handleImageGenerate()
  │           │     │     └── imageGenerationTool.handle(textMessage) → 生成图片
  │           │     │
  │           │     └── VOICE_REPLY → handleTextWithFileReply()
  │           │           ├── chatTool.handle() → LLM 对话
  │           │           ├── voiceService.synthesize() → TTS
  │           │           └── return WechatReply.file(bytes, "AI语音回复.mp3")
  │
  └── sendReply → 根据 reply 类型发送
```

## 五、助手回复存入上下文的完整路径

### 核心原则

- **用户消息**在路由之前由 `WechatMessageService.saveMessageToContext()` 统一存储
- **助手回复**由各个 Tool 自行存储，位置不统一

### 1. ChatTool — 最标准的路径

```
ChatTool.handle(message)
  └── chatService.chat(userId, text)
        ├── contextStore.getHistory(userId, 20)
        ├── llmClient.chat(userId, text, history)
        ├── contextStore.append(userId, "assistant", reply)  ← ✅ 在这里存
        └── return reply
```

源码 (`ChatService.chat()`)：
```java
String reply = llmClient.chat(userId, message, history);
contextStore.append(userId, "assistant", reply);
return reply;
```

### 2. VisionTool — 图片分析回复

```
VisionTool.handle(message)
  ├── downloadImageAsDataUrl()
  ├── visionService.analyze(dataUrl, null)
  ├── contextStore.append(userId, "user", "[用户发送了一张图片]")  ← 补一条用户标记
  ├── contextStore.append(userId, "assistant", reply)              ← ✅ 存回复
  └── return WechatReply.text(reply)
```

特殊之处：图片消息到达时 `saveMessageToContext` 已经存了 `[图片]`+CDN参数，但 VisionTool 又补了一条纯文本标记 `[用户发送了一张图片]`，上下文里有两条用户记录。

### 3. ImageGenerationTool — 图片生成回复

```
ImageGenerationTool.handle(message)
  ├── enrichPrompt()
  ├── imageGenerationService.generate(prompt) → 拿到URL
  ├── contextStore.updateLastMediaUrl(userId, "[图片]", imageUrl)
  ├── imageClient.downloadImage(imageUrl)
  ├── contextStore.append(userId, "assistant", "[图片]", null, null, imageUrl)  ← ✅ 存回复
  └── return WechatReply.image(imageBytes)
```

特殊之处：存的是 `[图片]` 标记 + imageUrl，后续 VisionTool 能从上下文里找到这张图片的 URL 重新下载分析。

### 4. VoiceTool — 语音回复（最复杂）

```
VoiceTool.handle(message)
  ├── getVoiceText() → ASR转文字
  ├── voiceService.classifyVoiceIntent() → 意图分类
  └── routeByIntent()
        ├── CHAT → handleChatWithTts()
        │     ├── chatTool.handle() → ChatService.chat() 内部已存 assistant reply ✅
        │     ├── TTS 成功:
        │     │     ├── voiceService.synthesize(replyText)
        │     │     ├── contextStore.append(userId, "assistant", "[本条回复以MP3文件形式发送]")  ← ✅ 覆盖存
        │     │     └── return WechatReply.file(bytes, "AI语音回复.mp3")
        │     └── TTS 失败 → 降级文本（ChatService 已存过）
        │
        ├── IMAGE_GENERATE → imageGenerationTool.handle() 内部已存 ✅
        └── VOICE_REPLY → handleTextWithFileReply() 类似 CHAT+TTS
```

关键问题：CHAT 意图 + TTS 成功时，上下文里会有**两条** assistant 记录：
1. `ChatService` 存的原文回复 `"今天天气很好..."`
2. `VoiceTool` 覆盖存的 `"[本条回复以MP3文件形式发送]"`

### 5. FileTool — 文件分析回复

```
FileTool.handle(message)
  ├── A. 图片文件路径:
  │     ├── visionService.analyze(dataUrl, "请详细描述这张图片")
  │     ├── contextStore.append(userId, "user", "[文件图片: name]\n" + description)
  │     └── return WechatReply.text(description)
  │     ⚠️ 没有存 assistant 回复！
  │
  └── B. 文档文件路径:
        ├── fileParseService.parse() → 提取文本+内嵌图片
        ├── contextStore.append(userId, "user", "用户发送了文件《name》\n内容...")
        ├── chatService.chat() → ChatService 内部存 assistant reply ✅
        └── return WechatReply.text(analysis)
```

### 6. FileGenerationTool — 文件生成回复

```
FileGenerationTool.handle(message)
  ├── fileGenerationService.generate(userId, text)
  ├── contextStore.append(userId, "assistant", "[已生成文件: xxx.pdf]")  ← ✅ 存回复
  └── return WechatReply.file(bytes, "xxx.pdf", description)
```

### 存回复位置对比

| 处理器 | 谁存 | 存什么 | 存在哪 |
|--------|------|--------|--------|
| ChatTool | `ChatService.chat()` | LLM 原文回复 | LLM 调用之后 |
| VisionTool | `VisionTool.handle()` | 分析文本 + 用户图片标记 | 分析完成之后 |
| ImageGenerationTool | `ImageGenerationTool.handle()` | `[图片]` + imageUrl | 下载图片之后 |
| VoiceTool (CHAT+TTS) | ChatService + VoiceTool | 原文 + "[MP3文件发送]" | 两处都存 |
| VoiceTool (IMG_GEN) | ImageGenerationTool | `[图片]` + imageUrl | 由 ImageGen 内部存 |
| FileTool (图片) | **仅用户侧** | `[文件图片]` + 描述 | ⚠️ assistant 未存 |
| FileTool (文档) | `ChatService.chat()` | LLM 分析结果 | 间接存 |
| FileGenerationTool | `FileGenerationTool.handle()` | `[已生成文件: name]` | 生成完成之后 |

### 存在的问题

1. **FileTool 图片分支**没有存 assistant 回复到上下文，后续对话缺少这段分析记录
2. **VoiceTool CHAT+TTS** 路径会存两条 assistant 记录（ChatService 原文 + VoiceTool 标记），语义重复
3. 各 Tool 存回复的位置和方式不统一，有的在 Service 层存，有的在 Tool 层存，维护时容易遗漏
