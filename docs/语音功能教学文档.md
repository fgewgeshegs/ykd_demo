## 1. 整体架构概览

语音功能采用经典的**分层架构**，从底层到顶层共 5 层：

```
┌─────────────────────────────────────────────────────┐
│  Layer 5: 消息发送层                                  │
│  WechatMessageService  →  WechatILinkClient          │
│  轮询消息、发送回复（文本/图片/文件）                    │
├─────────────────────────────────────────────────────┤
│  Layer 4: 路由层                                      │
│  MessageRouter                                       │
│  根据消息类型 + 意图 → 选择 Handler                     │
├─────────────────────────────────────────────────────┤
│  Layer 3: 消息处理层                                  │
│  VoiceHandler / AIChatHandler / ImageGenerationHandler│
│  业务编排：ASR → 意图分类 → 路由 → TTS                │
├─────────────────────────────────────────────────────┤
│  Layer 2: 业务逻辑层                                  │
│  VoiceService / ChatService / ImageGenerationService  │
│  重试机制、结果解析、意图分类                          │
├─────────────────────────────────────────────────────┤
│  Layer 1: API 调用层                                  │
│  VoiceClient / LLMClient / ImageClient               │
│  HTTP 调用阿里云 DashScope API                        │
├─────────────────────────────────────────────────────┤
│  Layer 0: 配置与数据模型                              │
│  VoiceProperties / WechatMessage / WechatReply / Intent│
└─────────────────────────────────────────────────────┘
```

**依赖方向**：上层依赖下层，下层不知道上层的存在。

## 2. 一条语音消息的完整旅程

下面以用户发送一条语音"帮我画一只猫"为例，追踪全链路：

```
用户微信发语音 "帮我画一只猫"
  │
  ▼
┌──────────────────────────────────────────────────────────┐
│ Step 1: WechatMessageService.pollLoop()                  │
│   定时轮询 wechatClient.receiveMessages()                 │
│   收到 WeixinMessage → 解析出 voice_item                  │
│   构建 WechatMessage {type=VOICE, voiceText="帮我画一只猫"}│
└──────────────────────────────────────────────────────────┘
  │
  ▼
┌──────────────────────────────────────────────────────────┐
│ Step 2: saveMessageToContext(wechatMsg)                   │
│   存入上下文: "[语音]帮我画一只猫"                          │
└──────────────────────────────────────────────────────────┘
  │
  ▼
┌──────────────────────────────────────────────────────────┐
│ Step 3: messageRouter.route(wechatMsg)                   │
│   发现 type == VOICE → 交给 voiceHandler.handle()        │
└──────────────────────────────────────────────────────────┘
  │
  ▼
┌──────────────────────────────────────────────────────────┐
│ Step 4: VoiceHandler.handle()                            │
│   ┌─ 4a. getVoiceText(): 获取 ASR 文本                   │
│   │   优先用微信自带识别结果（voiceText）                  │
│   │   若无则下载音频 → voiceService.transcribe()          │
│   │                                                      │
│   ┌─ 4b. voiceService.classifyVoiceIntent(text)          │
│   │   调用 LLM 判断：有意义? 什么意图?                    │
│   │   返回 {meaningful=true, intent=IMAGE_GENERATE}      │
│   │                                                      │
│   └─ 4c. routeByIntent(IMAGE_GENERATE, text, msg)        │
│       switch IMAGE_GENERATE → handleImageGenerate()      │
│         └─ imageGenerationHandler.handle(textMessage)    │
│              → 生成图片 → 返回 WechatReply.image(bytes)   │
└──────────────────────────────────────────────────────────┘
  │
  ▼
┌──────────────────────────────────────────────────────────┐
│ Step 5: sendReply(toUserId, reply)                       │
│   发现 reply.type == IMAGE                               │
│   → wechatClient.sendImageMessage(userId, bytes, fallback)│
│      异步上传 CDN → 发送图片给用户                         │
└──────────────────────────────────────────────────────────┘
```

**如果是 TTS 语音回复**（用户说"用语音回答我"），Step 4c 会走 `handleChatWithTts()`：

```
4c. handleChatWithTts(textMessage, ttsEnabled=true)
    ├─ chatHandler.handle() → 文字回复 "你好，有什么可以帮你？"
    ├─ voiceService.synthesize(replyText) → TTS 合成 MP3
    └─ return WechatReply.file(mp3Bytes, "AI语音回复.mp3", replyText)

Step 5: sendReply()
    → wechatClient.sendFileMessage() → 以文件形式发送 MP3
```

## 3. 逐层拆解：配置层

### 3.1 VoiceProperties — 语音配置

文件：`src/main/java/.../ai/llm/VoiceProperties.java`

```java
@Data
@Component
@ConfigurationProperties(prefix = "voice")  // 绑定 application.properties 中 voice.* 开头的配置
public class VoiceProperties {
    private String apiKey;              // DashScope API 密钥
    private String asrBaseUrl;          // ASR API 地址（默认：DashScope 转录服务）
    private String asrModel = "paraformer-v2";   // ASR 模型
    private String ttsBaseUrl;          // TTS API 地址（默认：百炼平台）
    private String ttsModel = "cosyvoice-v2";    // TTS 模型
    private String ttsVoice = "longxiaochun_v2"; // TTS 音色
    private boolean ttsEnabled = true;  // 是否启用 TTS（false 时语音消息文本回复）
}
```

**对应配置（application.properties）：**

```properties
voice.api-key=sk-xxxxxxxx
voice.asr-model=paraformer-v2
voice.tts-model=cosyvoice-v2
voice.tts-voice=longxiaochun_v2
voice.tts-enabled=true
```

**设计要点**：
- `@ConfigurationProperties(prefix = "voice")` 让 Spring Boot 自动把 `voice.api-key` 映射到 `apiKey` 字段
- 默认值避免必填项过多，开箱就能跑

## 4. 逐层拆解：API 调用层（VoiceClient）

文件：`src/main/java/.../ai/llm/VoiceClient.java`

这是最底层，直接和阿里云 DashScope API 打交道。

### 4.1 ASR（语音识别）：`asr(audioBytes, encodeType)`

```
输入：音频字节 + 编码类型（4=wav, 6=amr, 7=silk）
输出：识别出的文字

流程：
  ① 上传音频到 DashScope 文件服务 → 获得 fileUrl
  ② 调用 Paraformer 转录 API，传入 fileUrl
  ③ 解析响应：
     - 同步响应：直接拿 output.text
     - 异步响应：拿到 task_id，轮询直到 SUCCEEDED/FAILED
       （轮询策略：2s → 4s → 8s → 16s → 32s，最多 5 次）
```

**关键代码片段 — 构建请求体：**

```java
// 构建 Paraformer 请求
ObjectNode root = objectMapper.createObjectNode();
root.put("model", "paraformer-v2");

ObjectNode input = root.putObject("input");
ArrayNode fileUrls = input.putArray("file_urls");
fileUrls.add(fileUrl);         // 把刚上传的文件 URL 传进去

ObjectNode parameters = root.putObject("parameters");
parameters.put("sample_rate", 16000);
parameters.put("format", "wav");
```

**关键代码片段 — 解析异步响应：**

```java
// 如果 output.task_status 存在 → 异步任务
if ("PENDING".equals(taskStatus) || "RUNNING".equals(taskStatus)) {
    return pollAsyncTask(taskId);  // 去轮询
}
```

### 4.2 TTS（语音合成）：`tts(text)`

```
输入：文字
输出：MP3 音频字节数组

流程：
  ① 构建 CosyVoice 请求体
  ② POST 到 TTS API
  ③ 从响应中提取 output.audio.url（云端音频地址）
  ④ 下载音频字节 → 返回
```

**TTS 请求体结构：**

```java
root.put("model", "cosyvoice-v2");
ObjectNode input = root.putObject("input");
input.put("text", text);              // 要合成的文字
input.put("voice", "longxiaochun_v2"); // 音色
input.put("format", "mp3");
input.put("sample_rate", 24000);
```

### 4.3 辅助方法

| 方法 | 作用 |
|------|------|
| `parseSampleRate(bytes)` | 解析 WAV 头部的采样率（field offset 24），非 WAV 默认 8000Hz |
| `parsePlaytime(bytes, textHint)` | 解析 WAV 头部计算时长。回退策略：文本长度估算 → 文件大小估算 |
| `uploadFile(bytes, fileName)` | 上传音频到 DashScope 文件服务（multipart/form-data） |
| `downloadAudio(url)` | 从 URL 下载音频字节 |

**parsePlaytime 回退策略（重要设计模式）：**

```java
// 策略1：解析 WAV 头部（最准确）
if (是 WAV 格式) → 解析 dataSize / (sampleRate × channels × bytesPerSample) × 1000

// 策略2：文本长度估算（中文约 3-4 字/秒）
int durationMs = textHint.length() * 300;  // 每字 300ms

// 策略3：文件大小估算（均值 8KB/s）
int estimatedMs = audioBytes.length * 1000 / 8000;
```

### 4.4 VoiceClientException

文件：`src/main/java/.../ai/voice/VoiceClientException.java`

```java
@Getter
public class VoiceClientException extends RuntimeException {
    private final String errorCode;  // API 返回的错误码，如 "DataInspectionFailed"
}
```

**设计意图**：自定义异常携带 `errorCode`，让上层（VoiceService）可以区分错误类型做不同重试策略。

## 5. 逐层拆解：业务逻辑层（VoiceService）

文件：`src/main/java/.../ai/voice/VoiceService.java`

在 VoiceClient 之上封装了**重试、解析、意图分类**等业务逻辑。

### 5.1 `transcribe(audioBytes, encodeType)` — ASR 含重试

```java
public String transcribe(byte[] audioBytes, int encodeType) {
    int attempt = 0;
    while (attempt < MAX_RETRIES) {  // 最多重试 3 次
        try {
            String text = voiceClient.asr(audioBytes, encodeType);
            if (text != null && !text.trim().isEmpty()) return text.trim();
            return null;  // 空文本直接返回 null，不重试
        } catch (VoiceClientException e) {
            attempt++;
            if (attempt >= MAX_RETRIES) return null;  // 重试耗尽
            Thread.sleep(2000L * attempt);            // 退避：2s → 4s → 6s
        } catch (Exception e) {
            return null;  // 非业务异常直接放弃
        }
    }
    return null;
}
```

**设计要点**：
- `VoiceClientException`（业务错误）才重试，`Exception`（网络异常等）直接放弃
- 线性退避：2s → 4s → 6s
- 空文本直接返回 null，不浪费重试次数

### 5.2 `synthesize(text)` — TTS 含重试

逻辑和 `transcribe` 完全相同，差别在于：
- 调用的是 `voiceClient.tts(text)`
- 返回值是 `VoiceSynthesisResult`（包含字节、时长、采样率、URL）

### 5.3 `classifyVoiceIntent(voiceText)` — 语音意图分类

```java
public VoiceIntentResult classifyVoiceIntent(String voiceText) {
    // 1. 调用 LLM（带 voice-system-prompt.txt 作为系统提示词）
    String rawResult = llmClient.chatWithSystemPrompt(voiceIntentPrompt, voiceText);

    // 2. 解析 JSON 返回
    return parseVoiceIntentResult(rawResult);
}
```

**LLM 返回格式：**
```json
有含义：{"meaningful": true, "intent": "CHAT"}
无含义：{"meaningful": false}
```

**解析逻辑（parseVoiceIntentResult）：**

```java
// 1. 从回复中提取 JSON 对象（容错：LLM 可能多输出文字）
int start = json.indexOf('{');
int end = json.lastIndexOf('}');
String jsonStr = json.substring(start, end + 1);

// 2. 用 Jackson ObjectMapper 解析
JsonNode root = objectMapper.readTree(jsonStr);
boolean meaningful = root.get("meaningful").asBoolean();

// 3. 解析 intent → Intent 枚举
Intent intent = Intent.valueOf(intentNode.asText());  // "CHAT" → Intent.CHAT
```

**兜底策略**：任何异常都返回 `{meaningful: true, intent: CHAT}`（宁可回复也不要没反应）

### 5.4 内部类

```java
// 意图分类结果
public static class VoiceIntentResult {
    boolean meaningful;  // 是否有实际含义（false → 回复"没听清"）
    Intent intent;       // 意图（仅 meaningful=true 时有效）
}

// TTS 合成结果
public static class VoiceSynthesisResult {
    byte[] audioBytes;   // MP3 音频字节
    int playtimeMs;      // 时长（毫秒）
    int encodeType;      // 编码类型（固定 4 = WAV）
    int sampleRate;      // 采样率（Hz）
    String audioUrl;     // 云端音频 URL（存上下文，可复用）
}
```

## 6. 逐层拆解：消息处理层（VoiceHandler）

文件：`src/main/java/.../wechat/handler/VoiceHandler.java`

这是整个语音流程的**编排中心**。

### 6.1 入口：`handle(message)`

```java
public WechatReply handle(WechatMessage message) {
    if (message.getType() != MessageType.VOICE) return null;  // 不是我管的

    // Step 1: 获取 ASR 文本
    String voiceText = getVoiceText(message);
    if (voiceText == null) return WechatReply.text("抱歉，语音识别失败");

    // Step 2: 判断是否有意义 + 什么意图
    VoiceIntentResult result = voiceService.classifyVoiceIntent(voiceText);
    if (!result.isMeaningful()) return WechatReply.text("抱歉，我没听清");

    // Step 3: 按意图路由
    return routeByIntent(result.getIntent(), voiceText, message);
}
```

### 6.2 `getVoiceText(message)` — 双路径 ASR

```
├── 路径1（优先）：微信服务端自带 ASR
│   message.voiceText 不为空 → 直接用，零成本
│
└── 路径2（降级）：自行下载音频 + Paraformer 识别
    wechatClient.downloadMedia(encryptParam, aesKey) → 下载音频字节
    voiceService.transcribe(audioBytes, encodeType)  → Paraformer 识别
```

### 6.3 `routeByIntent(intent, voiceText, msg)` — 意图分发

```java
private WechatReply routeByIntent(Intent intent, String voiceText, WechatMessage original) {
    WechatMessage textMessage = buildTextMessage(voiceText, original);  // 构建伪 TEXT 消息

    return switch (intent) {
        case IMAGE_GENERATE  → handleImageGenerate(textMessage);
        case VOICE_REPLY     → handleTextWithFileReply(textMessage);
        case CHAT            → handleChatWithTts(textMessage, isTtsEnabled());
        default              → 兜底文本
    };
}
```

**为什么构建伪 TEXT 消息？**

下游 Handler（AIChatHandler、ImageGenerationHandler）是按 `MessageType.TEXT` 设计的。语音 ASR 后得到的文本，需要包装成 TEXT 消息才能喂给它们。

### 6.4 三条意图分支

```
                    VoiceHandler.routeByIntent()
                              │
          ┌───────────────────┼───────────────────┐
          ▼                   ▼                   ▼
   IMAGE_GENERATE        VOICE_REPLY            CHAT
          │                   │                   │
          ▼                   ▼                   ▼
handleImageGenerate   handleTextWithFileReply  handleChatWithTts
          │                   │                   │
          ▼                   ▼                   ▼
ImageGenerationHandler   chatHandler.handle()  chatHandler.handle()
   .handle()                 ↓                     ↓
          ↓            voiceService           voiceService
返回图片/文本兜底     .synthesize()           .synthesize()
                          ↓                 (if ttsEnabled)
                     返回 MP3 文件               ↓
                                          返回 MP3 文件
                                          (或纯文本 if !ttsEnabled)
```

**三个分支的区别**：

| 分支 | 触发条件 | 处理方式 | 返回类型 |
|------|---------|---------|---------|
| IMAGE_GENERATE | "帮我画一只猫" | 调图片生成服务 | IMAGE 或 TEXT（兜底） |
| VOICE_REPLY | "用语音回答我" | LLM 对话 → TTS | FILE（MP3） |
| CHAT | 普通聊天 | LLM 对话 → (可选 TTS) | FILE（MP3）或 TEXT |

### 6.5 回复类型的历史变迁

```
旧版（已删除）：
  CHAT + TTS 启用 → WechatReply.voice(bytes, playtime, encodeType, sampleRate, text)
  → 微信"语音气泡"形式发送（类似微信自带语音消息）

新版（当前）：
  CHAT + TTS 启用 → WechatReply.file(bytes, "AI语音回复.mp3", text)
  → 微信"文件"形式发送（MP3 文件）
```

**为什么改？** 语音气泡发送路径不稳定/不可用，文件形式更可靠。

## 7. 逐层拆解：路由层（MessageRouter）

文件：`src/main/java/.../wechat/MessageRouter.java`

### 7.1 核心逻辑：`route(message)`

```java
public WechatReply route(WechatMessage message) {
    // 图片消息 → 直接走 VisionHandler
    if (message.getType() == MessageType.IMAGE) {
        return visionHandler.handle(message);
    }

    // 语音消息 → 直接走 VoiceHandler
    if (message.getType() == MessageType.VOICE) {
        return voiceHandler.handle(message);
    }

    // 文本消息 → 先识别意图，再按意图路由
    if (message.getType() == MessageType.TEXT) {
        Intent intent = intentClassifier.classify(message.getText());

        if (intent == Intent.VOICE_REPLY) {
            // 文本消息 + 语音回复意图 → VoiceHandler 的 TTS 路径
            return voiceHandler.handleTextWithFileReply(message);
        }

        MessageHandler target = selectHandler(intent);
        return target.handle(message);
    }

    // 未知类型 → 兜底
    return fallbackHandler.handle(message);
}
```

**设计要点**：
- 图片和语音消息**不经过意图分类**，直接交给对应 Handler
- 文本消息先分类意图再路由
- `VOICE_REPLY` 意图很特殊：它是**文本消息**进来，分类后发现用户想要语音回复，于是走 VoiceHandler

### 7.2 `selectHandler(intent)` — 意图到 Handler 映射

```java
CHAT           → AIChatHandler
IMAGE_GENERATE → ImageGenerationHandler
IMAGE_ANALYZE  → VisionHandler
其他           → SimpleReplyHandler（兜底："暂时无法理解该消息类型"）
```

## 8. 逐层拆解：发送层

### 8.1 WechatMessageService — 消息收发协调

文件：`src/main/java/.../wechat/service/WechatMessageService.java`

**消息轮询**（`pollLoop`）：
```
while (running) {
    List<WeixinMessage> messages = wechatClient.receiveMessages();  // 收消息
    for (each message) {
        WechatMessage wechatMsg = buildWechatMessage(item);  // 1. 解析
        saveMessageToContext(wechatMsg);                      // 2. 存上下文
        WechatReply reply = messageRouter.route(wechatMsg);  // 3. 路由处理
        sendReply(reply);                                     // 4. 发送回复
    }
    sleep(pollIntervalMs);  // 间隔再轮询
}
```

**发送回复**（`sendReply`）：
```java
switch (reply.getType()) {
    case IMAGE → wechatClient.sendImageMessage(userId, bytes, "发送失败兜底文字");
    case FILE  → wechatClient.sendFileMessage(userId, bytes, fileName, description);
    default    → wechatClient.sendTextMessage(userId, text);
}
```

**上下文存储**（`saveMessageToContext`）：
```java
switch (msg.getType()) {
    case TEXT  → contextStore.append(userId, "user", text);
    case VOICE → contextStore.append(userId, "user", "[语音]" + voiceText, ...);
    case IMAGE → contextStore.append(userId, "user", "[图片]", ...);
}
```

### 8.2 WechatILinkClient — 微信 SDK 封装

文件：`src/main/java/.../wechat/client/WechatILinkClient.java`

| 方法 | 用途 | 同步/异步 |
|------|------|-----------|
| `sendTextMessage(userId, text)` | 发送文字 | 同步 |
| `sendImageMessage(userId, bytes, fallback)` | 发送图片，失败降级文字 | 异步 + 重试2次 |
| `sendFileMessage(userId, bytes, name, desc)` | 发送文件（MP3等） | 异步 |
| `startTyping(userId)` | 显示"正在输入" | 同步 |
| `receiveMessages()` | 拉取新消息 | 同步 |
| `downloadMedia(param, key)` | 下载 CDN 媒体 | 同步 |

**为什么语音/图片/文件要用异步发送？**

微信消息轮询线程是**单线程**的。如果 TTS 合成 + CDN 上传耗时 5 秒，同步发送会阻塞整个消息处理，导致其他消息排队等待。异步发送后消息处理线程立即释放，继续处理下一条。

```java
// 异步模式
CompletableFuture.runAsync(() -> {
    client.sendImage(toUserId, bytesCopy, "image.jpg", "");
});
```

## 9. 数据模型一览

### 9.1 消息类型枚举

```java
public enum MessageType {
    TEXT,     // 文本
    IMAGE,    // 图片
    VOICE,    // 语音（仅用于接收，回复不使用）
    VIDEO,    // 视频（预留）
    FILE      // 文件（用于 TTS 回复的 MP3）
}
```

### 9.4 WechatReply（回复模型）

```java
@Data
@Accessors(chain = true)  // 支持链式调用
public class WechatReply {
    MessageType type;
    String text;           // TEXT 类型时有效
    byte[] imageBytes;     // IMAGE 类型时有效
    byte[] fileBytes;      // FILE 类型时有效
    String fileName;       // FILE 文件名
    String fileDescription; // FILE 描述

    // 工厂方法
    static WechatReply text(String text);
    static WechatReply image(byte[] imageBytes);
    static WechatReply file(byte[] fileBytes, String fileName, String description);

    boolean hasContent();  // 判空
}
```

**为什么用工厂方法而不是构造器？**

1. 工厂方法名（`text`/`image`/`file`）直接表达意图，比 `new WechatReply(type=TEXT, text=...)` 更可读
2. 链式调用 `.setXxx()` 配合 `@Accessors(chain = true)` 让构建更流畅

## 总结：关键设计决策

1. **为什么 ASR 优先用微信自带结果？**
   微信服务端已经做了语音识别（免费且快速），只有当它返回空时才下载音频自己调 Paraformer。

2. **为什么意图分类要同时判断"有意义"和"意图"？**
   一次 LLM 调用完成两件事，省一次 API 调用。`meaningful=false` 时直接回复"没听清"，不再走后续路由。

3. **为什么语音用异步发送？**
   消息处理是单线程轮询，同步发送会阻塞后续消息。异步发送后立即释放线程。

4. **为什么 VOICE_REPLY 是文本消息进来的？**
   用户可能打字说"用语音回复我"，而非发语音。这时候需要识别文本意图，然后走 TTS 路径。

5. **为什么废弃语音气泡改用文件？**
   微信 iLink SDK 的语音气泡 API 不稳定/不可用，文件形式（MP3）更可靠。
