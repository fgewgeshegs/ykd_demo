# 语音功能深度解析

> 本文档基于项目代码，逐步拆解语音消息从接收到回复的完整链路。

---

## 一、核心概念

### ASR 和 TTS 是什么？

LLM **只认文字，不认语音**。所以中间需要两个"翻译"：

| 缩写 | 全称 | 作用 | 类比 |
|------|------|------|------|
| ASR | Automatic Speech Recognition | 语音 → 文字 | 你听别人说话，听懂意思 |
| TTS | Text-to-Speech | 文字 → 语音 | 你开口说话，把想法说出来 |

```
你的语音 → ASR（听懂你）→ 文字 → LLM（思考）→ 文字 → TTS（开口回答）→ 语音还给你
```

### 三种语音文件格式

微信语音消息支持三种音频编码格式：

| encodeType | 格式 | 说明 | 类比 |
|---|---|---|---|
| 4 | WAV | 无损原始录音，音质最好，文件最大（1分钟 ≈ 10MB） | 拍照用 RAW 原图 |
| 6 | AMR | 专为语音设计的压缩格式，手机打电话用的（1分钟 ≈ 1MB） | 手机通话的声音 |
| 7 | SILK | 微信自家格式，Skype 团队开发，微信语音条就是这个 | 微信语音条的音质 |

**实际场景：** 微信发语音大多是 SILK（7），WAV 和 AMR 比较少见。

格式只在一个地方起作用——当微信 ASR 结果为空，需要自己下载音频调 DashScope Paraformer 时，要告诉外部 API 这是什么格式。微信自带的 ASR 不挑格式，服务器端已经处理好了。

## 二、语音流程总览

### 生活类比

你去餐厅，**不会说普通话**，只会说方言：

```
你（方言）→ 服务员A（翻译方言→普通话）→ 厨师（做菜）→ 服务员B（翻译普通话→方言）→ 你（听到方言回答）
```

对应到项目：

```
你发语音 → ASR（语音→文字）→ LLM（思考回答）→ TTS（文字→语音）→ 你收到语音
```

### 为什么需要绕这一圈？

```
LLM 的世界：输入必须是文字 → 输出也是文字
你的世界：  输入是语音 ← 输出也想听语音

所以中间需要两个翻译：
  ASR：你的语音 → 文字（喂给 LLM）
  TTS：LLM 的文字 → 语音（还给你）
```

## 三、逐步详解

### 第一步：消息进来 —— WechatMessageService 接收

微信消息不是你主动去拉的，是 `WechatMessageService` 启动后一直在**轮询**（默认每 3 秒一次）。

```java
// WechatMessageService.java
@PostConstruct
public void start() {
    if (!wechatProperties.isEnabled()) return;  // 开关控制
    // 等微信登录完成（最多等60秒）
    // 登录成功后，启动轮询线程
    pollThread = new Thread(this::pollLoop, "wechat-poll-thread");
    pollThread.start();
}
```

### 第二步：轮询拿到消息 —— pollLoop

```java
// WechatMessageService.java
private void pollLoop() {
    while (running.get()) {
        List<WeixinMessage> messages = wechatClient.receiveMessages();
        for (WeixinMessage msg : messages) {
            for (var item : msg.getItem_list()) {
                WechatMessage wechatMsg = buildWechatMessage(item, ...);
                saveMessageToContext(wechatMsg);       // 存入对话历史
                wechatClient.startTyping(fromUserId);  // 显示"对方正在输入..."
                WechatReply reply = messageRouter.route(wechatMsg);  // 路由分发
                sendReply(fromUserId, reply);          // 发送回复
            }
        }
        Thread.sleep(3000);  // 3秒后再拉
    }
}
```

主线：`receiveMessages()` → `buildWechatMessage()` → `saveMessageToContext()` → `messageRouter.route()` → `sendReply()`

### 第三步：识别语音消息 —— buildWechatMessage

```java
// WechatMessageService.java
if (item.getVoice_item() != null) {
    var voice = item.getVoice_item();
    wechatMsg.setType(MessageType.VOICE);
    wechatMsg.setVoiceText(voice.getText());          // 微信服务端 ASR 结果（可能为空）
    wechatMsg.setPlaytime(voice.getPlaytime());       // 播放时长
    wechatMsg.setEncodeType(voice.getEncode_type());  // 编码类型(4=WAV, 6=AMR, 7=SILK)
    wechatMsg.setVoiceSampleRate(voice.getSample_rate());
    if (voice.getMedia() != null) {
        wechatMsg.setVoiceEncryptQueryParam(voice.getMedia().getEncrypt_query_param());
        wechatMsg.setVoiceAesKey(voice.getMedia().getAes_key());
    }
}
```

微信语音消息带了两个关键信息：
- `voiceText` —— 微信自己做的语音识别结果（可能有，可能没有）
- `voiceEncryptQueryParam` + `voiceAesKey` —— 用来从微信 CDN 下载原始音频文件

### 第四步：存入上下文 —— saveMessageToContext

```java
// WechatMessageService.java
case VOICE -> {
    String vText = msg.getVoiceText() != null ? msg.getVoiceText() : "";
    String content = !vText.isEmpty() ? "[语音]" + vText : "[语音消息]";
    contextStore.append(userId, "user", content, vEnc, vKey, null);
    // 不只存文字，还存了 CDN 下载参数，后续可以重新下载音频
}
```

`Message` 是个 record：

```java
public record Message(
    String role,              // "user" 或 "assistant"
    String content,           // "[语音]今天天气怎么样"
    String mediaEncryptParam, // CDN 加密参数
    String mediaAesKey,       // 解密密钥
    String mediaUrl           // 媒体 URL（语音场景为 null）
) {}
```

### 第五步：路由分发 —— MessageRouter.route()

```java
// MessageRouter.java
public WechatReply route(WechatMessage message) {
    // 图片消息 → VisionTool
    if (message.getType() == MessageType.IMAGE) {
        return visionTool.handle(message);
    }
    // 语音消息 → VoiceTool（不做意图分类，VoiceTool 内部自己做）
    if (message.getType() == MessageType.VOICE) {
        return voiceTool.handle(message);
    }
    // 文本消息 → 先做意图分类，再路由
    if (message.getType() == MessageType.TEXT) {
        Intent intent = intentClassifier.classify(message.getText());
        // ...
    }
}
```

语音消息在这里**不走 IntentClassifier**，直接扔给 VoiceTool。因为语音有自己的意图分类逻辑，和文本的不一样。

## 四、ASR 阶段 —— 语音转文字

### 两层 ASR 降级策略

```java
// VoiceTool.java
private String getVoiceText(WechatMessage message) {
    // 第一层：优先用微信服务端的 ASR 结果（免费）
    if (message.getVoiceText() != null && !message.getVoiceText().trim().isEmpty()) {
        return message.getVoiceText().trim();
    }

    // 第二层：微信没给结果，下载音频调 DashScope Paraformer（收费）
    byte[] audioBytes = wechatClient.downloadMedia(
        message.getVoiceEncryptQueryParam(),
        message.getVoiceAesKey()
    );
    return voiceService.transcribe(audioBytes, encodeType);
}
```

| | 微信 SDK ASR | DashScope Paraformer |
|---|---|---|
| **来源** | 微信服务端，SDK 内置 | 阿里云 DashScope，自己调 |
| **是否额外调 API** | 不用，消息里自带 | 要，HTTP POST |
| **是否花钱** | 免费 | 按调用量计费 |
| **可用性** | 可能为空（微信不保证每次都给） | 自己控制，带重试 |
| **调用时机** | 优先 | 降级 |

### Paraformer ASR 详情

```java
// VoiceClient.java
public String asr(byte[] audioBytes, int encodeType) {
    // 第一步：上传音频到 DashScope 文件服务
    String fileName = determineFileName(encodeType);
    // 4 → "audio.wav", 6 → "audio.amr", 7 → "audio.silk"
    String fileUrl = uploadFile(audioBytes, fileName);

    // 第二步：调用 Paraformer 转录 API
    // POST dashscope.aliyuncs.com/.../transcription

    // 第三步：解析响应（可能是同步或异步）
    return parseAsrResponse(response.body());
}
```

异步响应需要轮询（指数退避：2s → 4s → 8s → 16s → 32s，最多 5 次）。

`VoiceService` 包了一层重试：最多 3 次，间隔递增（2s × attempt）。

## 五、意图分类阶段 —— 判断用户想干嘛

### 为什么语音不走文本的 IntentClassifier？

文本消息的 `IntentClassifier` 是一个通用分类器，只判断 CHAT / IMAGE_GENERATE / IMAGE_ANALYZE。

语音消息需要多判断一件事：**这段语音有没有意义？**（用户可能不小心按到语音按钮，发了 "嗯"、"啊"）

所以 VoiceTool 用了专门的 `VoiceService.classifyVoiceIntent()`，一次 LLM 调用完成两件事。

### 分类 Prompt

```
// prompts/voice-system-prompt.txt
你是一个语音消息理解和意图分类器。
你的任务：
1. 判断这段语音是否有实际含义（meaningful）
   - 无含义：背景噪音、口误、无意义的单字（"嗯"、"啊"、"呃"）
   - 有含义：任何有明确语义的表达

2. 如果有实际含义，判断意图：
   - CHAT: 普通聊天
   - IMAGE_GENERATE: 要求生成图片
   - VOICE_REPLY: 要求用语音回复

返回 JSON:
有含义: {"meaningful": true, "intent": "CHAT"}
无含义: {"meaningful": false}
```

### 解析逻辑

```java
// VoiceService.java
private VoiceIntentResult parseVoiceIntentResult(String json) {
    JsonNode root = objectMapper.readTree(json);

    boolean meaningful = root.get("meaningful").asBoolean();
    if (!meaningful) {
        return new VoiceIntentResult(false, null);
        // → VoiceTool 回复 "抱歉，我没听清"
    }

    String intentStr = root.get("intent").asText();  // "CHAT"
    Intent intent = Intent.valueOf(intentStr);
    return new VoiceIntentResult(true, intent);
}
```

### 意图汇总

| 用户说的话 | meaningful | intent | 处理方式 |
|---|---|---|---|
| "今天天气怎么样" | true | CHAT | 对话 + TTS 回语音 |
| "帮我画一只猫" | true | IMAGE_GENERATE | 生图，直接回图片 |
| "用语音回答我" | true | VOICE_REPLY | 对话 + TTS 回语音 |
| "嗯" | false | — | 回复"没听清" |

## 六、二次路由阶段 —— 按意图走不同路径

```java
// VoiceTool.java
private WechatReply routeByIntent(Intent intent, String voiceText, WechatMessage originalMessage) {
    WechatMessage textMessage = buildTextMessage(voiceText, originalMessage);
    // 把语音文本包装成一个"假的"文本消息

    return switch (intent) {
        case IMAGE_GENERATE -> handleImageGenerate(textMessage);
        // 用户说 "帮我画一只猫" → 图片生成

        case VOICE_REPLY -> handleTextWithFileReply(textMessage);
        // 用户说 "用语音回答我" → 文字对话 + TTS

        case CHAT -> handleChatWithTts(textMessage, voiceService.isTtsEnabled());
        // 用户说 "今天天气怎么样" → 文字对话 + 可选 TTS

        default -> WechatReply.text(FALLBACK_REPLY);
    };
}
```

### 为什么 CHAT 还要 TTS？

因为**你是用语音问的**。

```
你发语音问 → AI 语音回答（体验自然，像对话）
你发语音问 → AI 打字回答（感觉怪怪的，你都开口说话了，对方却打字回你）
```

**CHAT 是意图（你要聊天），TTS 是回复方式（用语音回你）。** 两回事。

- 意图 = 用户想干嘛 → 聊天
- 回复方式 = 用什么形式回 → 语音（因为你语音来的）

同理，如果用户**打字**进来走 CHAT，就不走 TTS，直接回文字：

```java
// 文本消息进来 → MessageRouter → ChatTool → 没有 TTS
// 语音消息进来 → VoiceTool → handleChatWithTts → 有 TTS
```

配置文件可以控制开关：`voice.tts-enabled=true`（开启）/ `false`（关闭，语音问的也回文字）。

### 三条路径详解

**路径一：CHAT（普通聊天，最常见）**

```
语音 "今天天气怎么样"
  → ASR → "今天天气怎么样"
  → 意图分类 → CHAT
  → ChatTool.handle() → 调 LLM → "今天北京晴，28°C"
  → TTS → CosyVoice 合成 MP3
  → 发送 MP3 文件
```

**路径二：IMAGE_GENERATE（生成图片）**

```
语音 "帮我画一只猫"
  → ASR → "帮我画一只猫"
  → 意图分类 → IMAGE_GENERATE
  → ImageGenerationTool.handle() → LLM 扩写 prompt → qwen-image-2.0 生图
  → 发送图片（不走 TTS，因为你要的是图片不是语音）
```

**路径三：VOICE_REPLY（明确要求语音回复）**

```
语音 "用语音告诉我今天天气"
  → ASR → "用语音告诉我今天天气"
  → 意图分类 → VOICE_REPLY
  → handleTextWithFileReply() → ChatTool → LLM 回答
  → TTS → 合成 MP3
  → 发送 MP3 文件
```

**无意义语音：**

```
语音 "嗯"
  → ASR → "嗯"
  → 意图分类 → meaningful=false
  → 回复 "抱歉，我没听清，请再说一遍。"
```

## 七、TTS 阶段 —— 文字转语音

### CosyVoice 合成

```java
// VoiceClient.java
public byte[] tts(String text) {
    // 构建请求:
    // {
    //   "model": "cosyvoice-v2",
    //   "input": {
    //     "text": "今天北京晴，28°C",
    //     "voice": "longxiaochun_v2",    ← 音色
    //     "format": "mp3",
    //     "sample_rate": 24000
    //   }
    // }
    // POST 到 DashScope TTS API
    // 响应里是一个下载 URL，不是音频本身
    // 下载音频字节 → 返回 byte[]
}
```

### 时长计算

```java
// VoiceClient.java - parsePlaytime()
// WAV 文件 → 解析 RIFF 头部，精确计算
// 非 WAV → 回退估算：
//   有文本 → 文本长度 × 300ms/字
//   无文本 → 文件大小 / 8KB/s
```

### 重试机制

```java
// VoiceService.java
public VoiceSynthesisResult synthesize(String text) {
    int attempt = 0;
    while (attempt < 3) {  // 最多重试3次
        try {
            byte[] audioBytes = voiceClient.tts(text);
            if (audioBytes != null) {
                return new VoiceSynthesisResult(audioBytes, playtimeMs, ...);
            }
        } catch (VoiceClientException e) {
            attempt++;
            Thread.sleep(2000L * attempt);  // 递增等待: 2s, 4s, 6s
        }
    }
    return null;  // 3次都失败 → 降级返回文字
}
```

## 八、特殊路径：文本消息触发语音回复

除了语音消息，还有一条路径——用户发**文字**但要求语音回复：

```
用户发文字: "用语音告诉我今天天气"
    │
    ▼
MessageRouter.route()
    → IntentClassifier.classify() → VOICE_REPLY
    → VoiceTool.handleTextWithFileReply(message)
        → ChatTool.handle() → 拿到文字回复
        → VoiceService.synthesize() → TTS
        → WechatReply.file(mp3)
```

这条路径不经过 ASR（因为用户发的是文字），但最终也走 TTS。

## 十一、完整调用链总结

```
WechatMessageService.pollLoop()              ← 轮询线程（每3秒）
  └→ buildWechatMessage()                    ← 识别为 VOICE 类型
  └→ saveMessageToContext()                  ← 存 "[语音]xxx" + CDN参数
  └→ MessageRouter.route()                   ← VOICE → VoiceTool
      └→ VoiceTool.handle()
          ├→ getVoiceText()                  ← ASR 阶段
          │   ├→ 微信 ASR（优先，免费）
          │   └→ Paraformer ASR（降级，收费）
          │
          ├→ voiceService.classifyVoiceIntent()  ← 意图分类
          │   └→ LLM 判断 meaningful + intent
          │       ├→ false → "没听清"
          │       └→ true  → CHAT / IMAGE_GENERATE / VOICE_REPLY
          │
          └→ routeByIntent()                 ← 二次路由
              ├→ CHAT → handleChatWithTts()
              │   ├→ chatTool.handle()       ← 调 LLM 对话
              │   └→ voiceService.synthesize()  ← TTS 合成
              │       └→ CosyVoice → MP3
              │
              ├→ IMAGE_GENERATE → handleImageGenerate()
              │   └→ imageGenerationTool.handle()  ← 生图
              │
              └→ VOICE_REPLY → handleTextWithFileReply()
                  ├→ chatTool.handle()       ← 调 LLM 对话
                  └→ voiceService.synthesize()  ← TTS 合成
  └→ sendReply()                             ← 发送到微信
      ├→ WechatReply.file → sendFileMessage()（MP3）
      ├→ WechatReply.image → sendImageMessage()（图片）
      └→ WechatReply.text → sendTextMessage()（文字）
```
