# 引用图片分析功能改造说明

## 问题

用户引用图片 + 打字"描述一下"，bot 回复"不知道你说什么"。

## 根因

```
用户引用图片 → 打字 "描述一下"
  ↓
pollLoop: item=TEXT, refMessage=IMAGE
  → extractRefMessage 设置 refImageEncryptParam ✅
  ↓
MessageRouter: type=TEXT → 意图分类 IMAGE_ANALYZE → 路由到 VisionHandler ✅
  ↓
VisionHandler.handle():
  if (type != IMAGE) → type=TEXT → return null ❌  ← 这里断了
```

VisionHandler 只接受 `IMAGE` 类型消息，但引用图片消息的类型是 `TEXT`。

## 修复

### VisionHandler.handle() 入口

```java
// 之前
if (message.getType() != MessageType.IMAGE) {
    return null;
}

// 之后
if (message.getType() == MessageType.IMAGE) {
    return analyzeImage(message);          // 直接图片
}
if (message.getType() == MessageType.TEXT && hasRefImage(message)) {
    return analyzeImage(message);          // 引用图片
}
return null;
```

### 图片下载优先级

`downloadImageAsDataUrl()` 改为三级降级：

```
① 消息直接附带的 CDN 参数（直接发图）
  → ② 引用消息中的 CDN 参数（引用图片）
    → ③ 图片 URL（最后兜底）
```

```java
private String downloadImageAsDataUrl(WechatMessage message) {
    // 优先：直接图片
    String encryptParam = message.getEncryptQueryParam();
    String aesKey = message.getAesKey();

    // 降级：引用图片
    if (isEmpty(encryptParam) || isEmpty(aesKey)) {
        encryptParam = message.getRefImageEncryptParam();
        aesKey = message.getRefImageAesKey();
    }

    // CDN 下载
    if (!isEmpty(encryptParam) && !isEmpty(aesKey)) {
        byte[] bytes = wechatClient.downloadMedia(encryptParam, aesKey);
        return "data:image/jpeg;base64," + Base64.getEncoder().encodeToString(bytes);
    }

    // 最后兜底：URL
    return message.getImageUrl();
}
```

### 图片参数保存

`saveImageParams()` 提取并存入 ContextStore，优先直接图片、降级引用图片：

```java
private void saveImageParams(WechatMessage message) {
    String ep = message.getEncryptQueryParam();
    String ak = message.getAesKey();
    if (isEmpty(ep)) { ep = message.getRefImageEncryptParam(); ak = message.getRefImageAesKey(); }
    if (!isEmpty(ep)) { contextStore.setLastImage(userId, ep, ak); }
}
```

### WechatMessageService 图文合并修复

引用图片时微信 SDK 可能发 `IMAGE+TEXT` 两个 item，refMessage 可能在 IMAGE item 上：

```java
// 之前：只在 TEXT item 上提取
extractRefMessage(textItem, wechatMsg);

// 之后：同时提取 IMAGE 和 TEXT item
extractRefMessage(item, wechatMsg);      // IMAGE item
extractRefMessage(textItem, wechatMsg);  // TEXT item
```

同时添加诊断日志，打印每个 item 的类型和 refMessage 状态：
```java
log.info("消息诊断 [{}/{}] | type=image={} text={} | ref={} | refImage={}", ...);
```

## 改动文件

| 文件 | 改动 |
|---|---|
| `VisionHandler.java` | 入口支持 TEXT+refImage、图片下载三级降级、参数保存提取 |
| `WechatMessageService.java` | 图文合并时同时提取两个 item 的 refMessage、添加诊断日志 |

## 流程

### 直接发图

```
用户发图片
  → MessageRouter: type=IMAGE → VisionHandler
  → analyzeImage()
    → downloadImageAsDataUrl: ① encryptQueryParam ✅
    → visionService.analyze()
    → saveImageParams → ContextStore
    → append → ContextStore
```

### 引用图片

```
用户引用图片 → 打字 "描述一下"
  → extractRefMessage: refImageEncryptParam ✅
  → MessageRouter: type=TEXT → IMAGE_ANALYZE → VisionHandler
  → hasRefImage(message) → true
  → analyzeImage()
    → downloadImageAsDataUrl: ① null → ② refImageEncryptParam ✅
    → visionService.analyze()
    → saveImageParams: ① null → ② refImageEncryptParam → ContextStore ✅
    → append → ContextStore ✅
  → 返回分析结果
```
