# Claw Assistant 项目包结构整理方案

> 日期：2026-07-17
> 分支：feature/image-generate

---

## 一、现状问题

### 1.1 AI 相关代码分散在根级别

```
ai/                ← chat, image, vision 业务能力
llm/               ← LLMClient, VisionClient, 配置, 提示词
intent/            ← 意图分类
```

`ai` 和 `llm`、`intent` 本质都是 AI 能力，却作为三个同级根包存在。`chat`/`image`/`vision` 是同级业务模块，但它们的配置（`ImageProperties` vs `LLMProperties`/`VisionProperties`）分在 `ai/config` 和 `llm/config` 两个包里。

### 1.2 weather 功能碎片化

一个天气功能横跨 4 个包：

| 文件 | 当前包 | 职责 |
|------|--------|------|
| `WeatherCommand` | `command` | 命令层 |
| `WeatherConfig` | `config` | 配置 |
| `WeatherTool` | `tool` | 业务层 |
| `WeatherResponse` | `model` | 数据模型 |

### 1.3 单文件薄包过多

| 包 | 文件数 | 说明 |
|----|--------|------|
| `exception` | 1 | `ClawException` |
| `router` | 1 | `MessageRouter`（且只被 wechat 使用）|
| `config` | 1 | `WeatherConfig` |

## 三、具体变更清单

### 3.1 文件移动

| 文件 | 从 | 到 |
|------|----|----|
| `ClawException.java` | `exception/` | 根包 |

| `LLMClient.java` | `llm/client/` | `ai/llm/` |
| `VisionClient.java` | `llm/client/` | `ai/llm/` |
| `LLMProperties.java` | `llm/config/` | `ai/llm/` |
| `VisionProperties.java` | `llm/config/` | `ai/llm/` |
| `SystemPromptProvider.java` | `llm/prompt/` | `ai/llm/` |
| `Intent.java` | `intent/` | `ai/classifier/` |

| `LLMIntentClassifier.java` | `intent/` | `ai/classifier/` |

### 3.2 空包删除

变更后以下包将清空，需删除目录：

- `exception/`
- `ai/config/`
- `llm/client/`
- `llm/config/`
- `llm/prompt/`
- `intent/`
- `config/`
- `model/`
- `tool/`
- `router/`
- `util/`

### 3.3 目录创建

- `ai/llm/`
- `ai/classifier/`
- `weather/`
- `common/`

### 3.4 import 更新

涉及文件修改的 16 个文件（被移动的文件）以及引用它们的文件。预计影响范围：

| 引用方 | 影响 |
|--------|------|
| `ChatService` → `ai.llm.LLMClient` | import 更新 |
| `VisionService` → `ai.llm.VisionClient` | import 更新 |
| `ImageGenerationService` → `ai.image.ImageProperties` | import 更新 |
| `MessageRouter` → `ai.classifier.*` | import 更新 |
| `AIChatHandler` + `VisionHandler` + `ImageGenerationHandler` → 对应 Service | import 更新 |
| `WeatherCommand` → `weather.WeatherTool` | import 更新 |

预计影响 **15~20 个文件的 import 语句**。

## 四、不变的部分

- `agent/` 包（7 个文件，死代码留作以后）
- 所有业务逻辑（Service 方法签名、Handler 签名、接口定义）
- 配置项前缀（`image.*`、`llm.*`、`wechat.*` 等）
- CLI 命令处理流程
- 微信消息处理流程

---

## 五、实施步骤

1. 创建新目录
2. 逐文件移动到新位置（git mv）
3. 逐文件修正 import
4. 删除空目录
5. `mvn clean compile` 验证编译通过
6. 启动应用、发送"生成小狗图片"验证端到端功能正常
7. 提交

---

## 六、风险与注意

- **git mv 保留文件历史**，不会丢失修改记录
- 移动后需在 IDE 中重新 import（IntelliJ 会自动优化）
- 已从 git 跟踪移除的 `application.properties` 不受影响
- 所有 Handler 的 `@Order` 已删除，路由不受包结构影响
