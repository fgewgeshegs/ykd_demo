
# 多轮对话上下文 & 微信引用消息改造教程

> 基于 `feature/image-generate` 最新代码，在 `cxx` 分支实现。

---

## 一、新增 context 包 — 会话上下文存储

### 为什么需要

之前 LLM 每次对话都是"失忆"的——用户说"刚才那张图呢"，bot 根本不知道"刚才"是什么。需要一个**按用户存储的对话历史**。

### 文件结构

```
context/
  Message.java           — 单条消息 record
  ContextStore.java      — 接口
  InMemoryContextStore.java — 内存实现
```

### 1.1 Message.java — 对话消息记录

```java
public record Message(String role, String content) {
}
```

`role` = `"user"` 或 `"assistant"`，直接对应 OpenAI 协议的 message role。

### 1.2 ContextStore.java — 存取接口

```java
public interface ContextStore {
    List<Message> getHistory(String userId, int maxMessages);  // 取最近 N 条
    void append(String userId, String role, String content);   // 追加一条
    void clear(String userId);                                  // 清除
}
```

### 1.3 InMemoryContextStore.java — 内存实现

```java
ConcurrentHashMap<String, Deque<Message>> store  // userId → 消息队列

getHistory: 取最后 N 条，返回 List.copyOf()（不可变快照）
append:    加到队尾，超出 50 条淘汰最早的
clear:     直接 remove
```

关键点：
- 用 `ConcurrentHashMap` + `synchronized (deque)` 保证线程安全
- `List.copyOf()` 返回快照，避免并发修改问题
- 50 条上限防止内存泄漏

### 为什么用接口

当前用内存实现，将来换 Redis 只需实现 `ContextStore`，上层代码不用改。

## 二、LLMClient / ChatService — 多轮对话

### 改了什么

**LLMClient.java** — 新增带历史的重载：

```java
// 新增：带历史的对话
public String chat(String userId, String text, List<Message> history) {
    return callLLM(systemPrompt, text, history);
}

// 原有：单轮对话，内部委托到上面
public String chat(String userId, String text) {
    return chat(userId, text, List.of());
}
```

**buildRequestBody** 的变化：

```java
// 之前
messages: [
  { role: "system", content: "..." },
  { role: "user",    content: "当前消息" }
]

// 之后
messages: [
  { role: "system",    content: "..." },
  { role: "user",      content: "你好" },         // ← 历史
  { role: "assistant", content: "你好！" },        // ← 历史
  { role: "user",      content: "刚才说的是什么" }  // ← 当前
]
```

**ChatService.java** — 注入 ContextStore，调用前后读写：

```java
public String chat(String userId, String message) {
    // 1. 取历史
    List<Message> history = contextStore.getHistory(userId, 20);

    // 2. 调 LLM（带历史）
    String reply = llmClient.chat(userId, message, history);

    // 3. 写回上下文
    contextStore.append(userId, "user", message);
    contextStore.append(userId, "assistant", reply);

    return reply;
}
```

## 三、VisionHandler — 图片分析写入上下文

### 之前

分析完图片返回文字就结束了，用户追问"图里的颜色"之类的问题，LLM 不知道之前分析过什么。

### 之后

```java
// 分析完成后写入上下文
contextStore.append(userId, "user", "[用户发送了一张图片]");
contextStore.append(userId, "assistant", reply);
```

这样后续文字对话的历史里就会有：
```
user: [用户发送了一张图片]
assistant: 这是一张包含...的图片...
user: 刚才图里的颜色是什么？
```

LLM 就能答出来了。

## 四、微信引用消息 — extractRefMessage

### 微信 SDK 的数据结构

用户长按消息 → 引用 → 打字发送后，SDK 返回的 `MessageItemDto` 里有个 `getRefMessage()`：

```java
RefMessage {
    title: "xxx",                    // 降级文本
    messageItem: MessageItemDto {    // 被引用的消息
        text: "...",                 // 文字内容
        image: ImageContent {        // 图片 CDN 参数
            encryptQueryParam, aesKey
        },
        voice: VoiceContent {        // 语音转写
            text: "..."
        }
    }
}
```

### WechatMessage 新增字段

```java
private String refMessageText;         // 引用消息的文字
private String refImageEncryptParam;   // 引用图片的 CDN 参数
private String refImageAesKey;         // 引用图片的解密密钥
private boolean hasAttachedImage;      // 图文合并标记
```

### extractRefMessage 逻辑

```java
void extractRefMessage(MessageItemDto item, WechatMessage wechatMsg) {
    RefMessage ref = item.getRefMessage();
    if (ref == null) return;

    MessageItemDto refItem = ref.getMessageItem();

    // 1. 图片参数（为后续编辑做准备）
    if (refItem.getImage() != null) {
        wechatMsg.setRefImageEncryptParam(refItem.getImage().getEncryptQueryParam());
        wechatMsg.setRefImageAesKey(refItem.getImage().getAesKey());
    }

    // 2. 文字内容（优先级：文本 > 语音转写 > title）
    if (refItem.getText() != null) {
        wechatMsg.setRefMessageText(refItem.getText());
    } else if (refItem.getVoice()?.getText() != null) {
        wechatMsg.setRefMessageText(refItem.getVoice().getText());
    } else if (ref.getTitle() != null) {
        wechatMsg.setRefMessageText(ref.getTitle());
    }
}
```

### AIChatHandler 拼入上下文

```java
String buildFullText(WechatMessage message) {
    String refText = message.getRefMessageText();
    if (refText == null) return message.getText();
    return "[用户引用了: \"" + refText + "\"]\n" + message.getText();
}
```

效果：用户引用"今天天气怎么样"→ 打字"用英文回答"，LLM 收到：
```
[用户引用了: "今天天气怎么样"]
用英文回答
```

## 五、图文合并消息 — pollLoop 改造

### 场景

用户在微信里选一张图 + 配文字一起发。

### SDK 数据结构

```java
itemList: [
    MessageItemDto { type: IMAGE, image: {...} },
    MessageItemDto { type: TEXT,  text: "这是什么" }
]
```

### pollLoop 改造

从 `forEach` 改为 indexed `for` 循环，检测相邻的 IMAGE+TEXT 或 TEXT+IMAGE：

```java
for (int i = 0; i < items.size(); i++) {
    // IMAGE + TEXT 合并
    if (item.isImage() && i+1 是 TEXT) {
        msg = buildImageMessage(item);        // 图片参数
        msg.setHasAttachedImage(true);        // 标记
        msg.setType(TEXT);                    // 类型是文字
        msg.setText(textItem.getText());      // 文字内容
        i++;  // 跳过下一条
    }
    // TEXT + IMAGE 同理...

    extractRefMessage(item, msg);  // 提取引用消息
    routeAndReply(msg);
}
```

### MessageRouter 路由

```java
if (message.getType() == IMAGE) {
    if (message.isHasAttachedImage()) {
        // 图文合并 → 按文字意图分类
        Intent intent = intentClassifier.classify(message.getText());
        handler = selectHandler(intent);
    } else {
        // 纯图片 → VisionHandler
        handler = visionHandler;
    }
}
```

## 六、关键设计决策

| 决策 | 理由 |
|---|---|
| ContextStore 用接口 | 将来可换 Redis/DB，不改上层 |
| 历史消息上限 50 条 | 防止内存泄漏，50 条足够覆盖大多数对话 |
| `List.copyOf()` 快照 | 避免 synchronized 块外并发修改 |
| buildFullText 前缀格式 | LLM 能理解"用户在引用某条消息" |
| refImageEncryptParam 预留 | 为后续图片编辑功能做准备（用户引用图片+编辑指令） |
