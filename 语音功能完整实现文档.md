# 语音功能完整实现文档

## 一、整体流程

```
═══════════════════════════════════════════════════════════════
用户按住说话 → 微信发来语音消息
═══════════════════════════════════════════════════════════════

① WechatMessageService.pollLoop()
   轮询收到消息 → item.getVoice_item() 提取语音数据
   → 构建 WechatMessage(type=VOICE, voiceText, voiceEncryptQueryParam,
                         voiceAesKey, playtime, encodeType, voiceSampleRate)

② saveMessageToContext(wechatMsg)
   统一存入上下文（发一条存一条）
   → VOICE: contextStore.addVoiceMedia(userId, enc, key, text)
            contextStore.append(userId, "user", "[语音]" + text)
   → TEXT:  contextStore.append(userId, "user", text)
   → IMAGE: contextStore.addImageMedia(userId, enc, key, url)
            contextStore.append(userId, "user", "[图片]")

③ MessageRouter.route(message)
   type == VOICE → VoiceHandler.handle(message)

④ VoiceHandler.handle()
   ├─ getVoiceText()
   │   ├─ voiceText 非空 → 直接用微信服务端转写（免费、即时）
   │   └─ voiceText 为空 → downloadMedia() → VoiceClient.asr()
   │       └─ 上传音频到 DashScope 文件服务 → Paraformer v2 转录
   │           └─ 同步响应直接返回，异步响应轮询（2s/4s/8s/16s/32s）
   │
   ├─ voiceService.classifyVoiceIntent(text)
   │   └─ LLM 一次调用完成两件事：
   │       ├─ meaningful?（过滤"嗯"/"啊"/噪音）
   │       └─ intent: CHAT / IMAGE_GENERATE / VOICE_REPLY
   │
   └─ routeByIntent(intent)
       ├─ IMAGE_GENERATE → ImageGenerationHandler → 生成图片
       ├─ VOICE_REPLY   → AIChatHandler → ChatService → LLM 对话
       │                  → VoiceService.synthesize() → CosyVoice TTS
       │                  → WechatReply.file(mp3, "AI语音回复.mp3", text)
       │                  → sendFileMessage() → 用户收到 MP3 文件
       └─ CHAT          → AIChatHandler → ChatService → LLM 对话
                          → 可选 TTS → WechatReply.voice()
                          → sendVoiceMessage() → 用户收到语音气泡

⑤ sendReply(toUserId, reply)
   ├─ VOICE → wechatClient.sendVoiceMessage() 异步发送，失败降级文字
   ├─ FILE  → wechatClient.sendFileMessage()   异步发送
   ├─ IMAGE → wechatClient.sendImageMessage()  异步发送
   └─ TEXT  → wechatClient.sendTextMessage()   同步发送

⑥ 用户看到/听到回复（语音气泡 / MP3文件 / 文字）
```

## 二、意图分类体系

### 2.1 文字消息 —— LLMIntentClassifier

专门 Prompt 让 LLM 判断意图，返回 JSON：

```json
{"intent": "CHAT"}           // 普通对话
{"intent": "IMAGE_GENERATE"} // 生成图片
{"intent": "IMAGE_ANALYZE"}  // 分析图片
{"intent": "VOICE_REPLY"}    // 用户要求语音回复（"用语音告诉我"）
```

### 2.2 语音消息 —— VoiceService.classifyVoiceIntent()

独立 Prompt（`voice-system-prompt.txt`），比文字分类多做一步"有意义判断"：

```json
// 有意义
{"meaningful": true, "intent": "CHAT"}
// 无意义（噪音 / "嗯" / "啊"）
{"meaningful": false}
```

无意义时直接回复"抱歉，我没听清，请再说一遍"，不浪费 LLM 对话资源。

### 2.3 路由分发 —— MessageRouter

| 消息类型 | 意图 | 处理器 | 回复方式 |
|---------|------|--------|---------|
| IMAGE | — | VisionHandler | 文字 |
| VOICE | CHAT | VoiceHandler → AIChatHandler | 文字 + 可选语音气泡 |
| VOICE | IMAGE_GENERATE | VoiceHandler → ImageGenerationHandler | 图片 |
| VOICE | VOICE_REPLY | VoiceHandler → AIChatHandler → TTS | MP3 文件 |
| TEXT | CHAT | AIChatHandler | 文字 |
| TEXT | IMAGE_GENERATE | ImageGenerationHandler | 图片 |
| TEXT | IMAGE_ANALYZE | VisionHandler | 文字 |
| TEXT | VOICE_REPLY | VoiceHandler.handleTextWithFileReply() | MP3 文件 |

## 三、语音 API 对接（VoiceClient）

### 3.1 ASR —— Paraformer v2

```
端点: https://dashscope.aliyuncs.com/api/v1/services/audio/transcription/transcription
鉴权: Authorization: Bearer {voice.api-key}

流程:
  ① POST https://dashscope.aliyuncs.com/api/v1/files
     上传音频（multipart/form-data）
     → 获取 file_url
  ② POST 转录请求
     {"model":"paraformer-v2", "input":{"file_urls":["..."]},
      "parameters":{"sample_rate":16000, "format":"wav"}}
  ③ 同步响应: {"output":{"text":"识别结果"}}
     异步响应: {"output":{"task_status":"PENDING", "task_id":"..."}}
     → 轮询 GET /api/v1/tasks/{task_id}
     → 指数退避: 2s / 4s / 8s / 16s / 32s（最多 5 次）
```

### 3.2 TTS —— CosyVoice v3-flash

```
端点: https://dashscope.aliyuncs.com/api/v1/services/audio/speech/synthesis
鉴权: Authorization: Bearer {voice.api-key}

请求:
  {"model":"cosyvoice-v3-flash",
   "input":{"text":"要合成的文字", "voice":"longanyang",
            "format":"mp3", "sample_rate":24000}}

响应:
  {"output":{"audio":{"url":"https://...", "expires_at":1772697707}}}
  → GET url → 下载 MP3 字节

播放时长计算:
  ① 尝试 WAV RIFF 头解析（dataSize / byteRate）
  ② 回退文字长度估算（中文 ~280ms/字）
  ③ 回退文件大小估算（~8KB/s）
  ④ clamp(100ms, 10min)
```

### 3.3 语音服务封装（VoiceService）

- `transcribe(byte[], encodeType)`: ASR 最多重试 3 次，指数退避
- `synthesize(text)`: TTS 最多重试 3 次，返回 `VoiceSynthesisResult(bytes, playtimeMs, encodeType, sampleRate)`
- `classifyVoiceIntent(text)`: LLM 语音意图分类，返回 `VoiceIntentResult(meaningful, intent)`
- `isTtsEnabled()`: 读取 `voice.tts-enabled` 配置开关

## 四、回复类型体系（WechatReply）

| 类型 | 工厂方法 | 字段 | 发送方式 |
|------|---------|------|---------|
| TEXT | `WechatReply.text(str)` | text | `sendTextMessage()` 同步 |
| IMAGE | `WechatReply.image(bytes)` | imageBytes | `sendImageMessage()` 异步 |
| VOICE | `WechatReply.voice(bytes, playtime, encodeType, sampleRate, fallback)` | voiceBytes, playtime, encodeType, sampleRate, textFallback | `sendVoiceMessage()` 异步 + 失败降级文字 |
| FILE | `WechatReply.file(bytes, name, desc)` | fileBytes, fileName, fileDescription | `sendFileMessage()` 异步 |

## 五、统一上下文存储

### 5.1 设计原则

所有消息（文字/语音/图片）统一存入一个队列，CDN 参数直接挂在 Message 记录上，不维护独立的媒体存储。

### 5.2 Message 数据结构

```java
public record Message(
    String role,              // "user" / "assistant"
    String content,           // 展示文本（"[语音]xxx" / "[图片]" / 原始文字）
    String mediaEncryptParam, // CDN 加密参数（语音/图片消息时有值）
    String mediaAesKey,       // CDN 解密密钥
    String mediaUrl           // 图片 URL
) {}
```

### 5.3 存储结构（InMemory / Redis 统一逻辑）

```
store: userId → Deque<Message>
  ├─ Message("user", "今天天气怎么样")
  ├─ Message("assistant", "今天北京晴，25°C")
  ├─ Message("user", "[语音]画一只猫",  enc, key, null)
  ├─ Message("assistant", "好的，正在为你生成...")
  ├─ Message("user", "[图片]",          enc, key, "https://...")
  └─ ...
```

- 最多保留 50 条/用户，超出自动淘汰最早的
- 发给 LLM 时取最近 20 条（`getHistory(userId, 20)`）

### 5.4 媒体查询

| 场景 | API | 实现 |
|------|-----|------|
| 最近一张图片 | `getLastImage(userId)` → [enc, key] | 反向扫描历史，找第一条 `[图片]` 且 hasMedia 的 Message |
| 最近一条语音 | `getLastVoice(userId)` → [enc, key, text] | 反向扫描历史，找第一条 `[语音]*` 且 hasMedia 的 Message |
| 所有语音 | `getVoiceMediaList(userId)` → List<[enc, key, text]> | 正向扫描所有 `[语音]*` 且 hasMedia 的 Message |
| 所有图片 | `getImageMediaList(userId)` → List<[enc, key, url]> | 正向扫描所有 `[图片]` 且 hasMedia 的 Message |

### 5.5 Redis 实现

```
Key:   ctx:{userId}:msgs  →  Redis LIST
Value: JSON 序列化的 Message

操作:
  append   → RPUSH + LTRIM (限长) + EXPIRE (TTL 7天)
  history  → LRANGE -N -1 (取最近 N 条)
  lastImage → LRANGE 全量 + 反向扫描
  clear    → DEL key
```

### 5.6 存储切换

```properties
# 内存（默认）
context.redis.enabled=false

# Redis
context.redis.enabled=true
context.redis.ttl-days=7
context.redis.max-messages=50
spring.data.redis.host=localhost
spring.data.redis.port=6379
```

Spring Bean 自动二选一：
- `context.redis.enabled=true` → `RedisContextStore`（`@ConditionalOnProperty(havingValue="true")`）
- 否则 → `InMemoryContextStore`（`@ConditionalOnProperty(matchIfMissing=true)`）
